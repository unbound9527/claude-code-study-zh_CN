# Claude Code 中文教程项目

## 项目概述

Claude Code 中文学习教程，7 个学习阶段，46 个 Markdown 文档。

## 目录结构

```
docs/
├── L0-环境配置/     # 安装、代理、国产模型
├── L1-认识工具/     # AI 基础、第一次对话、Git、CLI
├── L2-掌握用法/     # 清晰表达、工具使用、语音、Prompt
├── L3-熟练运用/     # 任务分解、上下文、迭代、工作流
├── L4-精通技巧/     # 记忆、子代理、Teams、MCP、Hooks

└── 附录/            # 模板库、FAQ、官方文档
```

## 文档规范

### 标题层级
- H1（#）：文档标题
- H2（##）：主要章节
- H3（###）：子章节
- 禁止跳过层级

### 链接规范
- 同章节：`02-first-chat.md`
- 跨章节：`../L2-掌握用法/01-clear-request.md`
- 外部链接：完整 URL

### 格式规范
- 分割线：`---`
- 列表：`-` 或 `1.` 保持一致
- 代码块：``` 并标注语言

## 可用 Skills

| 命令 | 用途 |
|------|------|
| `/quality [路径]` | 质量检查（链接、标题、格式） |
| `/polish <文件>` | 文档润色 |
| `/review [路径]` | 专业性核查 |
| `/fetch-info <URL>` | 从 URL 提取信息 |
| `/build` | 构建静态网站 |
| `/preview` | 本地预览 |

## 技术栈

- Claude Code CLI
- MCP - filesystem, github
- Hooks - preToolUse, postCommit
- Agent Teams - quality-check-team

## Git 规范

- 提交信息使用中文
- 提交前检查链接有效性
- 保持文档与 README 同步

## 项目约定

1. 文档风格：简洁、直接、不废话
2. 表格用于对比，不滥用
3. 示例要实用
4. 删除"本章目标"、"小测验"等冗余章节
5. 严禁编造虚假信息
