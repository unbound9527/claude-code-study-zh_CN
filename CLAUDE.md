## 文档规范

### 标题层级
- H1（#）：文档标题
- H2（##）：主要章节
- H3（###）：子章节
- 禁止跳过层级

### 链接规范
- 同章节：`02-first-chat.md`
- 跨章节：`../L2-掌握用法/01-clear-request.md`
- 外部链接：完整 URL
- 注意：文档已移至根目录，不再使用 `docs/` 前缀

### 格式规范
- 分割线：`---`
- 列表：`-` 或 `1.` 保持一致
- 代码块：``` 并标注语言

## Git 规范

- 提交信息使用中文
- 提交前检查链接有效性
- 保持文档与 README 同步

## 项目约定

1. 文档风格：简洁、直接、不废话
2. 表格用于对比，谨慎使用
3. 示例要实用
4. 删除"本章目标"、"小测验"等冗余章节
5. 严禁编造虚假信息
6. **内容深度标准：要么深挖到位，要么合并或删除**
   - 有独特见解、实战经验、原理剖析 → 独立成篇
   - 仅表面罗列常识、无干货 → 合并到相关章节或删除
   - 能用一页 PPT 讲完的内容不适合单独成篇
   - 写之前先问：这篇文章能让读者获得什么独特价值？
