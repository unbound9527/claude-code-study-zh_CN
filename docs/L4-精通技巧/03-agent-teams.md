# Agent Teams：组建专家团队

## 本章目标

- 理解什么是 Agent Teams
- 能组建不同"专家"的团队
- 知道何时用 Agent Teams 而非子代理

---

## 什么是 Agent Teams？

**Agent Teams = 组建专家顾问团**

就像现实中的项目需要不同专家（律师、会计师、设计师），Agent Teams 让你同时组建多个专业 AI，每个 AI 像一个专家，各自从自己的角度给你建议。

```
你需要做一个重要决策
       ↓
组建顾问团队
       ├── 市场专家 → 分析市场
       ├── 财务专家 → 分析资金
       └── 技术专家 → 分析可行性
       ↓
整合成完整建议
```

---

## 子代理 vs Agent Teams

| 特点 | 子代理 | Agent Teams |
|------|--------|-------------|
| 适合 | 同类任务并行 | 不同专家协作 |
| 比喻 | 一个人同时做多件事 | 一个团队开会 |
| 复杂度 | 简单 | 复杂 |

```
子代理 = 你让一个人同时做 PPT、写文案、想标题
Agent Teams = 你请设计师、文案、策略师一起开会
```

---

## 什么时候用 Agent Teams？

| 场景 | 需要的专家 |
|------|-----------|
| 创业决策 | 市场、财务、运营 |
| 买房决策 | 房产、律师、财务规划 |
| 职业规划 | 行业顾问、HR、职业教练 |
| 旅行规划 | 行程、预算、当地向导 |
| 产品开发 | 市场、技术、用户体验 |

---

## 怎么用？

### 句式

```
我要 [目标]，请组建专家团队：
- [专家 1]：负责 [职责]
- [专家 2]：负责 [职责]
- [专家 3]：负责 [职责]

让他们分别给我建议，最后整合成 [格式]
```

---

## 示例

### 示例：开奶茶店

```
我打算开一家奶茶店，请组建顾问团队：
- 市场分析师：分析附近竞争店铺、目标客户群
- 财务顾问：估算启动资金（装修、设备、原料）、每月固定成本、预期回本周期
- 运营专家：制定日常运营流程、人员配置建议
- 营销专家：设计开业活动和会员体系

每位专家给出专业建议，最后整合成一份完整的奶茶店创业计划书。
```

### 示例：职业转型

```
我想从传统行业转到互联网产品经理，请组建团队：
- 行业顾问：分析互联网行业现状、岗位需求
- HR 专家：评估我现有技能是否匹配，差距在哪里
- 职业教练：制定 6 个月转型计划
- 过来人：分享转型经验和常见坑

整合成一份转型路线图。
```

## Agent Teams 的协作模式

### 主从模式（推荐）

```
你（指挥官）
    └── 主代理（协调）
          ├── 专家 A → 负责领域 A
          ├── 专家 B → 负责领域 B
          └── 专家 C → 负责领域 C

主代理协调各方，最终向你汇报
```

### 对等模式（适合复杂）

```
专家 A ←——→ 专家 B
    ↕              ↕
专家 C ←——→ 专家 D

专家们相互讨论，最终达成共识
```

---

## 注意事项

| 原则 | 说明 |
|------|------|
| 3-5 个专家足够 | 不要太多，难以整合 |
| 职责要清晰 | 避免重叠 |
| 指定输出格式 | 最后要整合成完整文档 |
| 设定约束条件 | 如预算、时间限制 |

## 跟做

1. 假设你要做一次重要的旅行规划
2. 用 Agent Teams 方式提问：
```
我要去日本旅游 7 天，预算 2 万，5 人（2 老 2 大 1 小），请组建顾问团队：
- 行程规划师
- 预算顾问
- 当地向导（推荐适合老人和小孩的）

整合成一份详细的旅行计划
```
