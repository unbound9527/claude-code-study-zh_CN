# 安全最佳实践

## 本章目标

- 了解使用 Claude Code 的安全风险
- 掌握保护隐私和安全的技巧
- 能安全地使用 Claude Code

---

## 安全风险概览

| 风险类型 | 可能的危害 |
|----------|------------|
| **隐私泄露** | 密码、密钥、个人信息被暴露 |
| **数据安全** | 敏感文件被读取或篡改 |
| **权限滥用** | 未授权的操作被执行 |
| **社交工程** | 被诱导执行危险操作 |

---

## 敏感信息不要告诉 AI

### ❌ 绝对不能告诉 Claude Code 的内容

| 类型 | 示例 | 后果 |
|------|------|------|
| **密码** | 数据库密码、邮箱密码 | 被盗用 |
| **API 密钥** | AWS keys、支付密钥 | 被滥用，产生费用 |
| **银行卡信息** | 卡号、密码、验证码 | 金融诈骗 |
| **身份证信息** | 身份证号、照片 | 身份盗用 |
| **商业机密** | 核心技术、客户数据 | 竞争对手获取 |

### ✅ 正确做法

```
❌ 不好：
"帮我查一下我的密码是啥，密码在文档里：[粘贴密码]"

✅ 好：
"帮我设计一个安全的密码策略"
```

## 文件操作安全

### 保护重要文件

1. **确认文件路径**
   ```
   AI：即将修改 /Users/me/Desktop/important.docx
   确认是否继续？ [y/n]
   ```

2. **设置禁止操作的目录**
   在 `settings.json` 中配置：
   ```json
   {
     "permissions": {
       "deny": [
         "rm -rf /",
         "/System",
         "~/.ssh"
       ]
     }
   }
   ```

### 危险命令要警惕

| 命令 | 风险 | 建议 |
|------|------|------|
| `rm -rf` | 不可恢复的删除 | 仔细确认路径 |
| `git push --force` | 覆盖远程历史 | 一般不要用 |
| `curl` 粘贴敏感信息 | 信息被记录 | 避免这样做 |

## API 密钥安全

### 环境变量

```
❌ 直接写在代码里：
API_KEY = "sk-xxxxx123456"

✅ 使用环境变量：
API_KEY = process.env.API_KEY
```

### 密钥管理工具

| 工具 | 说明 |
|------|------|
| **1Password** | 密码管理器 |
| **AWS Secrets Manager** | 云服务密钥管理 |
| **GitHub Secrets** | CI/CD 密钥管理 |

### Claude Code 中的密钥

```bash
# 不要这样用
claude
> 帮我用这个密钥调用 API：sk-xxxxx

# 正确做法
export API_KEY="sk-xxxxx"  # 在终端设置环境变量
claude
> 帮我调用 API（不提供密钥）
```

## 网络安全

### 避免在公共网络使用

| 网络环境 | 风险 |
|----------|------|
| 家庭/办公室网络 | 低风险 |
| 咖啡店 WiFi | 中风险，可能被监听 |
| 公共 WiFi | 高风险，不建议 |

### 使用代理时的注意

```bash
# 设置代理
export HTTPS_PROXY="http://proxy.example.com:8080"

# 但不要在代理日志中留下敏感信息
```

## Git 操作安全

### 防止提交敏感文件

创建 `.gitignore`：

```
# 敏感文件
.env
*.key
credentials.json
secrets.yml

# 临时文件
*.log
*.tmp
.DS_Store
```

### 提交前检查

```bash
# 查看将要提交的内容
git diff --staged

# 确保没有敏感信息
```

## 权限管理

### 合理设置权限

```json
{
  "permissions": {
    "allow": [
      "Read",      // 读取文件
      "Write",     // 写入文件
      "Bash"       // 执行命令
    ],
    "deny": [
      "rm -rf /",           // 危险删除
      "git push --force",   // 强制推送
      "curl .*password.*"    // 包含密码的请求
    ]
  }
}
```

### 权限级别

| 级别 | 说明 | 适用场景 |
|------|------|----------|
| 严格 | 只读，无执行 | 学习和研究 |
| 中等 | 读写，无危险命令 | 日常使用 |
| 宽松 | 全部允许 | 可信赖的环境 |

## 安全 Checklist

使用 Claude Code 前检查：

- [ ] 不要告诉 AI 任何密码或密钥
- [ ] 重要文件做好备份
- [ ] 设置合理的权限限制
- [ ] 不在公共 WiFi 下处理敏感内容
- [ ] 使用 `.gitignore` 保护敏感文件
- [ ] 危险命令执行前仔细确认

## 发现安全问题怎么办？

| 情况 | 应对 |
|------|------|
| 不小心发送了密码 | 立即更换密码，撤销相关密钥 |
| 发现异常操作 | 停止 Claude Code，检查日志 |
| 需要紧急帮助 | 联系 Anthropic 支持 |

## 跟做

1. 检查你的 `~/.claude/settings.json` 权限配置
2. 确保 `.gitignore` 包含敏感文件类型
3. 确认没有在代码中硬编码密钥

---

## 练一练

1. 创建一个 `.gitignore` 文件，忽略 `.env`、`*.key`、`secrets.json`
2. 检查你常用的工作目录是否有敏感文件没被忽略
3. 配置 Claude Code 权限，禁止执行 `rm -rf /`
