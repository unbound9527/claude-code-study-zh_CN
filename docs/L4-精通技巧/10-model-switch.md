# 模型选择与切换

## 本章目标

- 了解 Claude Code 支持的模型
- 知道如何在不同场景选择合适的模型
- 能切换使用不同的模型

---

## Claude 支持哪些模型？

Claude Code 支持多个 AI 模型，不同模型有不同的特点：

| 模型 | 速度 | 智能水平 | 成本 | 适合场景 |
|------|------|----------|------|----------|
| **Claude Opus 4.6** | 慢 | 最高 | 高 | 复杂分析、深度推理 |
| **Claude Sonnet 4.6** | 中 | 高 | 中 | 日常任务、平衡之选 |
| **Claude Haiku 4.6** | 快 | 中 | 低 | 简单任务、快速问答 |

---

## 模型特点详解

### Claude Opus 4.6 - 最聪明

**优点**：
- 最强的理解和推理能力
- 能处理最复杂的任务
- 最好的代码理解和生成

**缺点**：
- 响应最慢
- 成本最高

**适合**：
- 复杂的代码重构
- 深入的技术分析
- 需要深度思考的问题

### Claude Sonnet 4.6 - 平衡之选

**优点**：
- 速度和智能的平衡
- 适合大多数日常任务
- 性价比高

**缺点**：
- 比 Opus 稍弱

**适合**：
- 写代码、做项目
- 回答问题
- 日常对话

### Claude Haiku 4.6 - 最快

**优点**：
- 响应最快
- 成本最低
- 即时反馈

**缺点**：
- 复杂任务表现较差

**适合**：
- 简单问答
- 快速翻译
- 格式化内容

## 切换模型的方法

### 方法一：在 settings.json 中设置默认模型

```json
{
  "model": "claude-opus-4-6"
}
```

### 方法二：对话中切换

```
/model claude-sonnet-4-6
```

### 方法三：指定特定任务使用不同模型

```
用 Opus 帮我分析这个架构问题：
[粘贴内容]

（然后切换回 Sonnet 做日常任务）
```

## 场景化模型选择

### 编程任务

| 任务类型 | 推荐模型 | 原因 |
|----------|----------|------|
| 写简单脚本 | Haiku | 快、够用 |
| 写完整功能 | Sonnet | 平衡 |
| 重构大型项目 | Opus | 需要最强的能力 |
| Debug 问题 | Sonnet | 通常够用 |

### 写作任务

| 任务类型 | 推荐模型 | 原因 |
|----------|----------|------|
| 写邮件 | Haiku | 简单直接 |
| 写文章 | Sonnet | 质量适中 |
| 写重要报告 | Opus | 需要高质量 |

### 学习任务

| 任务类型 | 推荐模型 | 原因 |
|----------|----------|------|
| 解释概念 | Sonnet | 够用且快速 |
| 深入研究 | Opus | 需要详细分析 |
| 出练习题 | Haiku | 简单直接 |

---

## 成本优化建议

### 按任务选模型

```
❌ 浪费：
问"今天吃什么"用 Opus
✅ 合理：
简单问题 → Haiku
日常任务 → Sonnet
复杂问题 → Opus
```

### 组合使用

```
工作流程：
1. 用 Haiku 快速问简单问题
2. 用 Sonnet 处理大部分任务
3. 只在真正需要时切换 Opus
```

## 模型切换命令

| 命令 | 作用 |
|------|------|
| `/model` | 查看当前模型 |
| `/model claude-opus-4-6` | 切换到 Opus |
| `/model claude-sonnet-4-6` | 切换到 Sonnet |
| `/model claude-haiku-4-6` | 切换到 Haiku |
| `/model auto` | 让 Claude Code 自动选择 |
