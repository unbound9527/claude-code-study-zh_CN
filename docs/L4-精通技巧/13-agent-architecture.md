## 什么是 Agent？

**Agent = 能自主决策和执行任务的 AI 系统**

普通的 AI 只是回答问题，而 Agent 能够：
- 感知环境（读取文件、获取网页）
- 做决策（判断下一步该做什么）
- 执行动作（写文件、执行命令）
- 循环迭代（直到完成任务）

```
Agent = 大脑（LLM）+ 手脚（Tools）+ 记忆（Memory）
```

---

## Agent 的工作方式

### 简单的问答

```
用户 → LLM → 回答
```

### Agent 模式

```
用户 → LLM → [决策：需要查文件] → 读取文件 → LLM → [决策：需要执行代码] → 执行 → LLM → 回答用户
```

---

## ReAct 循环

**ReAct = Reasoning + Acting（思考 + 行动）**

这是 Agent 的核心工作循环：

```
┌─────────────────────────────────────┐
│  1. Thought（思考）                  │
│     LLM 分析当前情况，决定下一步      │
├─────────────────────────────────────┤
│  2. Action（行动）                   │
│     执行工具（Read/Write/Bash...）    │
├─────────────────────────────────────┤
│  3. Observation（观察）               │
│     获取行动结果                      │
├─────────────────────────────────────┤
│  4. 循环直到完成任务                 │
│     如果没完成，回到步骤 1            │
└─────────────────────────────────────┘
```

### 具体示例

```
用户：帮我看看 src/app.js 有没有 bug

Thought：需要先读取文件内容
Action：Read(file="src/app.js")
Observation：文件内容已读取...

Thought：需要检查代码逻辑
Action：Bash(command="npm run lint src/app.js")
Observation：发现一个 lint 警告...

Thought：需要修复这个警告
Action：Edit(...)
Final：修复完成，向用户报告
```

## Tool-Use 循环

**Tool-Use = 使用工具完成任务**

```
用户问题
    ↓
LLM 判断是否需要工具
    ↓
[需要] → 选择工具 → 执行 → 获取结果 → 回到 LLM
    ↓
[不需要] → 直接回答
    ↓
用户得到答案
```

### Claude Code 的工具

| 工具 | 作用 | 例子 |
|------|------|------|
| Read | 读取文件 | "看一下这个文件" |
| Write | 写入文件 | "创建新文件" |
| Edit | 编辑文件 | "修改第三行" |
| Bash | 执行命令 | "运行测试" |
| Glob | 搜索文件 | "找所有 .js 文件" |
| Grep | 搜索内容 | "找包含 login 的代码" |
| WebFetch | 获取网页 | "查看这个网页" |
| Agent | 启动子代理 | "同时做三件事" |

## Agent 的决策过程

### LLM 在 Agent 中的角色

```
输入：用户问题 + 工具描述 + 历史记录
    ↓
LLM 分析：
- 用户要什么？
- 我需要什么工具？
- 当前情况是什么？
    ↓
输出：Thought + Action
```

### 决策的类型

| 决策 | 说明 |
|------|------|
| 是否使用工具 | 还是直接回答 |
| 使用哪个工具 | Read? Write? Bash? |
| 工具的参数 | 要读取哪个文件 |
| 何时停止 | 任务完成了吗 |

## Claude Code 的 Agent 实现

### 架构图

```
┌──────────────────────────────────────────┐
│              Claude Code                  │
├──────────────────────────────────────────┤
│  ┌────────────┐    ┌────────────┐       │
│  │   LLM      │◄───►│  Memory   │       │
│  │ (Claude)   │     │(上下文)   │       │
│  └─────┬──────┘    └────────────┘       │
│        │                                   │
│        ▼                                   │
│  ┌────────────────────────────────┐      │
│  │         Tool Executor           │      │
│  │  Read │ Write │ Bash │ Agent  │      │
│  └────────────────────────────────┘      │
└──────────────────────────────────────────┘
```

### 工作流程

```
1. 用户输入
       ↓
2. LLM 理解意图
       ↓
3. 判断需要哪些工具
       ↓
4. 执行工具序列
       ↓
5. 将结果反馈给 LLM
       ↓
6. 重复直到完成
       ↓
7. 返回最终答案
```

## 为什么需要循环？

### 单一任务

```
用户：2+2=？
LLM：直接回答 4
```

### 复杂任务

```
用户：帮我看看这个项目有没有 bug

LLM：需要先了解项目结构
  → 执行 Glob
  → 获取文件列表
  → LLM：需要检查代码
  → 执行 Read（多个文件）
  → LLM：发现问题
  → 执行 Edit 修复
  → LLM：修复完成
```

## Agent 的能力边界

| 方面 | 说明 | 如何扩展 |
|------|------|----------|
| 上下文限制 | 对话太长会遗忘早期内容 | 使用 Memory 存储长期记忆，或定期总结 |
| 工具可用性 | 内置工具有限 | 通过 MCP 协议扩展自定义工具 |
| 推理能力 | 复杂逻辑可能出错 | 分解任务、多步推理、迭代验证 |
| 幻觉 | 可能做出错误判断 | 要求提供来源、验证关键信息 |

**重要更新**：过去说 Agent "只能使用内置工具"已经不准确。通过 MCP（Model Context Protocol），Agent 可以连接任何外部服务——数据库、API、文件系统等，实现真正的"万物互联"。
