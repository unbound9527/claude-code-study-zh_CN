# 省钱技巧

## 本章目标

- 了解 Claude Code 的使用成本
- 学会降低使用成本的技巧
- 能在效果和成本之间找到平衡

---

## Claude Code 的计费方式

Claude Code 按 **Token** 计费：
- 输入的文字 → 消耗 Input Tokens
- AI 的回复 → 消耗 Output Tokens

**Token 约等于：1 个中文汉字 ≈ 1-2 个 Token**

---

## 省钱原则

| 原则 | 说明 |
|------|------|
| 精简问题 | 问清楚但不啰嗦 |
| 管理上下文 | 避免重复说同样的话 |
| 选择合适模型 | 不同任务用不同模型 |
| 利用缓存 | 减少重复请求 |

---

## 技巧一：精简提问

❌ 浪费版：
```
我是一个大学生，正在学习计算机科学专业，我现在有一道编程题不会做，
这道题是关于数据结构的，具体是关于二叉树的遍历问题，
我想请教你怎么解决这个问题，代码是用 Python 写的...
```

✅ 精简版：
```
用 Python 实现二叉树的中序遍历，包含注释。
```

**效果一样，但省 Token！**

---

## 技巧二：管理上下文

### 问题：对话太长会怎样？

```
Token 消耗 = 对话历史所有内容

对话 5 轮：消耗 100 tokens
对话 50 轮：消耗 1000 tokens
```

### 解决方法

| 方法 | 操作 |
|------|------|
| 定期总结 | 让 AI 总结要点，新对话继续 |
| /compact | 压缩历史记录 |
| 开新对话 | 话题转换时开新对话 |
| 简洁追问 | 不要重复之前说的 |

---

## 技巧三：选择合适的模型

Claude Code 支持多种模型：

| 模型 | 速度 | 成本 | 适合场景 |
|------|------|------|----------|
| Claude Opus | 慢 | 高 | 复杂分析、深度推理 |
| Claude Sonnet | 中 | 中 | 日常任务、平衡之选 |
| Claude Haiku | 快 | 低 | 简单任务、快速问答 |

### 省钱建议

```
❌ 过度使用：
问"今天吃什么"用 Opus 模型

✅ 合理选择：
- 简单问答 → Haiku
- 日常写作 → Sonnet
- 复杂分析 → Opus
```

---

## 技巧四：利用批量处理

❌ 低效：
```
每天单独问 AI：帮我写日报
```

✅ 高效：
```
周五下午一次性处理：
请根据我这周的工作内容，帮我生成周一到周五的 5 份日报模板
```

---

## 技巧五：避免无效迭代

❌ 浪费：
```
问：帮我写代码
AI：代码
问：重来
AI：新代码
问：还是不对
AI：新代码
...
```

✅ 高效：
```
问：帮我写一个 Python 函数，实现 [具体功能]，要求：
- 输入：[说明]
- 输出：[说明]
- 包含详细注释
```

---

## 技巧六：使用模板

把常用的 prompt 保存成模板，减少每次的输入量。

```
模板示例：
【解释概念】
用简单的话解释"[概念]"，举 2 个生活例子，不超过 100 字。
```

---

## 效果 vs 成本平衡

| 场景 | 建议的投入 |
|------|------------|
| 重要报告 | 不惜成本，追求质量 |
| 日常问答 | 用 Haiku，省钱快速 |
| 学习笔记 | 用 Sonnet，平衡之选 |
| 简单翻译 | 用 Haiku |

## 小测验

**1. Token 约等于什么？**
- A. 一个字母
- B. 一个汉字 ≈ 1-2 个 Token
- C. 一句话

**2. 什么模型成本最低？**
- A. Claude Opus
- B. Claude Sonnet
- C. Claude Haiku

**3. 精简提问能？**
- A. 没有效果
- B. 省 Token 就是省钱
- C. 让 AI 变慢

---

## 答案

1-B | 2-C | 3-B
