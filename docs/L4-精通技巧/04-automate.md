# 自动化工作流

## 本章目标

- 理解什么是自动化工作流
- 能设计简单的自动化流程
- 能用 Claude Code 提升日常效率

---

## 什么是自动化工作流？

**自动化 = 让重复的事自动完成**

想象你每天都要：
1. 整理昨天的笔记
2. 生成今日待办
3. 发送早安消息

**手动做**：每天花 30 分钟
**自动化后**：一键完成，节省时间

---

## 什么时候需要自动化？

| 场景 | 频率 | 自动化价值 |
|------|------|------------|
| 每天写同样的日志 | 每天 | 高 |
| 每周写同样的报告 | 每周 | 高 |
| 每次做类似的翻译 | 按需 | 中 |
| 偶尔写一次邮件 | 偶尔 | 低 |

**原则：重复越多，自动化价值越高**

## 常用自动化场景

### 1. 日/周报生成

```
模板：
请根据我提供的信息，生成一份 [日报/周报]：

[粘贴原始信息]

格式：
## 完成事项
## 进行中
## 下周计划
## 备注
```

### 2. 会议记录整理

```
模板：
请整理以下会议记录，提取：
1. 讨论的主要问题
2. 作出的决定
3. 待办事项（负责人 + 截止时间）
4. 下次会议时间

[粘贴会议记录]
```

### 3. 学习笔记整理

```
模板：
请把以下笔记整理成复习资料：
1. 提取核心概念
2. 标注重难点
3. 生成问答对
4. 出 3 道练习题

[粘贴笔记]
```

### 4. 批量翻译

```
模板：
请翻译以下内容为 [目标语言]，保持格式：

[粘贴内容]
```

## 设计你的工作流

### 步骤一：识别重复任务

问自己：
- 这件事我多久做一次？
- 每次需要多长时间？
- 步骤是否类似？

### 步骤二：提取不变的部分

```
每天写日报
├── 变：今天做了什么
├── 不变：格式结构
└── 自动化：填入内容 → 输出格式日报
```

### 步骤三：创建模板

把不变的部分做成模板，减少每次的输入量。

---

## 省力技巧

### 1. 保存常用模板

把模板保存到：
- 备忘录
- 笔记软件
- 快捷短语工具

### 2. 批量处理

```
❌ 分散处理：
每天单独问 AI 帮我写日报

✅ 批量处理：
1. 周五下午一次性整理一周的内容
2. 让 AI 一次生成 5 天的日报模板
```

### 3. 预设角色

```
我是一名高中数学老师，请以后按这个身份帮我：
- 出练习题
- 改作业
- 写教案
```

## 小测验

**1. 什么任务最适合自动化？**
- A. 只做一次的事
- B. 重复性高的任务
- C. 复杂的事

**2. 自动化工作流的核心是？**
- A. 让 AI 帮你做决定
- B. 把重复的步骤变成模板
- C. 完全不用动手

**3. 日报生成自动化的价值是？**
- A. 低（偶尔写一次）
- B. 中（每周一次）
- C. 高（每天都要写）

---

## 答案

1-B | 2-B | 3-C
