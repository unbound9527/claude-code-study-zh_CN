Git

Git 规范

黄金法则

独立分支、原子提交、规范信息、人工必审、全程追溯

分支规范

  • feat/ai-collab-功能名
  • fix/ai-collab-issue号
  • 禁止 AI 直接操作 main

提交规范

  • Conventional Commits
  • 必须标注 (via Claude Code)
  • 原子提交 ≤50 行改动

PR 模板

  • 变更说明 + 关联 Issue
  • 测试清单（100% 人工审核）
  • Claude 会话 ID
团队

团队规范

必存目录

  • .claude/skills/ — 团队共享技能
  • .claude/commands/ — 自定义斜杠命令
  • .claude/hooks/ — 强制自动化规则
  • .mcp.json — 团队 MCP 配置
  • .github/ — PR 模板/CI/CD

Explore-Plan-Code-Commit

强制 4 步流程

  • Explore — 只读扫描
  • Plan — 生成方案、人工审核
  • Code — 原子修改、自动测试
  • Commit — 规范提交、人工必审
省钱

省钱最佳实践

上下文控制

  • CLAUDE.md ≤150 行
  • 每 5-6 轮 /compact
  • 一次一会话，切换 /clear

效果：省 40-88%

文件过滤

  • .claudeignore 屏蔽
  • node_modules/dist/.env
  • 手动精准加载

效果：省 60%+

模型分层

  • Haiku — 简单任务
  • Sonnet — 业务开发
  • Opus — 仅必要时

效果：省 40-80%

公式：精简 CLAUDE.md + .claudeignore + 任务拆分 + 模型分层 + 前缀缓存

工作流

自动化工作流

四大自动化基石

  • Hooks — 管强制（事件触发）
  • Skills — 管流程（模块化 SOP）
  • MCP — 管外部（连接真实世界）
  • Commands — 管快捷（一键自动化）

代理循环

执行 → 测试 → 失败 → 修复 → 再测试 → 通过 → 提交

效果：缺陷率降低 60%

MCP+Skill 自动化闭环

Skill（SOP）→ MCP（执行）→ Hook（验证）→ 闭环

必配 Hooks

  • PreToolUse — 安全检查
  • PostToolUse — 自动格式化
  • Stop — 通知
  • PreCompact — 防止遗忘
TDD

TDD 测试驱动开发

红-绿-重构

  1. 红 — 写测试，运行确认失败
  2. 绿 — 写实现，测试通过
  3. 重构 — 优化代码

强制约束

  • 禁止 mock 不存在代码
  • 禁止修改已提交测试
  • 每步运行测试报告结果

质量门禁

  • 覆盖率 ≥85%
  • 测试通过 + lint 无错误
  • 格式合规

Hooks 自动化

  • onStop — 自动测试
  • onCommit — 强制验证

效果：AI 代码缺陷率降低 60%+
