# Bug 修复

## 修复流程

```
1. 描述问题（预期 vs 实际）
2. 提供完整错误信息
3. 提供最小复现代码
4. AI 分析原因
5. 确认方案后修改
6. 验证修复
```

## 如何描述问题

| 要素 | 说明 | 示例 |
|------|------|------|
| 预期 | 应该怎样 | 输入 [1,2,3] 应返回 6 |
| 实际 | 发生了什么 | 返回 None |
| 错误信息 | 完整的报错 | TypeError: ... |
| 位置 | 文件/行号 | main.py 第 23 行 |

## 常用句式

```
我的代码报错：
- 预期：[描述]
- 实际：[描述]
- 错误：[完整错误信息]
- 位置：[文件:行号]

代码：
[粘贴最小复现代码]

请分析原因并修复。
```

## 最小复现代码

| ✅ 好的复现 | ❌ 太长的复现 |
|-------------|--------------|
| 只保留出错部分 | 整个文件 |
| 简单数据 | 复杂对象 |
| 可独立运行 | 依赖外部模块 |

## 修复后验证

```
1. 让 AI 解释修复逻辑
2. 确认理解修复原因
3. 在本地运行测试
4. 确认原问题已解决
5. 检查是否引入新问题
```

## 常见 Bug 类型

| 类型 | 示例 | 解决方法 |
|------|------|----------|
| 空指针 | `None.foo` | 添加空值检查 |
| 数组越界 | `arr[100]` 超长度 | 边界检查 |
| 类型错误 | 字符串当数字用 | 类型转换 |
| 异步问题 | 回调顺序错 | async/await |
