# 常见陷阱与避坑

## 沟通类陷阱

| 陷阱 | 避坑 |
|------|------|
| 需求太模糊 | 用"做X，输出Y格式，关注Z"结构 |
| 一次说太多 | 分步进行，完成一步确认一步 |
| 没说明角色场景 | 明确读者、提供风格示例 |

## 技术类陷阱

| 陷阱 | 避坑 |
|------|------|
| 直接跑 AI 代码 | 先 review，用测试环境验证 |
| 不理解就复制 | 多问"为什么"，记录原理 |
| 忽视安全警告 | 认真阅读，必要时拒绝执行 |

## 效率类陷阱

| 陷阱 | 避坑 |
|------|------|
| 上下文过长不清理 | 用 /compact 或开新对话 |
| 简单问题用 AI | 简单语法直接查文档 |

## 操作类陷阱

| 陷阱 | 避坑 |
|------|------|
| 没确认就继续 | 每步让 AI 列出改动，确认后再继续 |
| 忽略 AI 建议 | 不轻易跳过，问清原因 |

## 问题发生后

```
1. 停止：不要再执行
2. 回滚：git checkout 恢复
3. 沟通：清楚描述问题
4. 预防：思考如何避免
```

## 避坑清单

```
□ 需求描述清楚了吗？
□ 会不会一次说太多？
□ 重要操作有备份吗？
□ 理解代码再运行吗？
□ 上下文太长了吗？
□ AI 的警告看了吗？
```
