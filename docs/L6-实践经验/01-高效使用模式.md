# 高效使用模式

## 任务分解

| 原则 | 说明 |
|------|------|
| 单一职责 | 每步只做一件事 |
| 有输出物 | 每步有明确交付 |
| 可验证 | 完成可检查 |

**分解示例**：
```
太粗："做网站"
↓ 分解
细一些："商品列表 + 详情页 + 购物车 + 结账"
```

## 提示词技巧

| 技巧 | 示例 |
|------|------|
| 明确输出 | "格式：表格，包含列名..." |
| 提供上下文 | "这是一个 Python 函数，用于..." |
| 指定角色 | "作为10年经验的产品经理..." |
| 分步引导 | "第一步...第二步..." |
| 给示例 | "像这样的风格：..." |

## 上下文管理

| 策略 | 适用 |
|------|------|
| 及时确认 | 每步完成后确认 |
| 定期总结 | "目前完成了什么？" |
| /compact | 对话变长时压缩 |
| 开新对话 | 任务相关但复杂 |

## 高效模板

**新项目启动**：
```
1. 描述项目目标
2. 说明技术栈约束
3. AI 生成项目结构
4. 确认后逐步实现
```

**代码审查**：
```
1. 提供代码："审查 src/auth.py"
2. 指定关注："重点关注安全性"
3. 获取问题清单
```

**Bug 修复**：
```
1. 描述预期 vs 实际
2. 提供完整错误信息
3. 提供最小复现代码
4. AI 分析原因
5. 确认后修改
```
