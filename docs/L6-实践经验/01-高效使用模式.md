## 任务分解最佳实践

### 为什么需要分解？

```
复杂任务："帮我做一个网站"
    ↓ 一次说太多
结果：AI 只做了表面，质量不稳定

分解后：
1. "帮我设计网站结构"
2. "写首页 HTML"
3. "写样式 CSS"
4. "添加交互 JS"
    ↓ 逐步完成
结果：每步都可控、可调整
```

### 分解原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 单一职责 | 每步只做一件事 | "写登录表单" 而非 "登录+注册+验证" |
| 有输出物 | 每步有明确交付 | "输出 README.md" 而非 "了解一下" |
| 可验证 | 完成可检查 | "运行测试" 而非 "确保能用" |

### 分解粒度参考

```
太粗："写一个电商网站"
    ↓ 建议分解

细一些："实现以下功能：
1. 商品列表页
2. 商品详情页
3. 购物车功能
4. 结账流程"

更细（如果 AI 容易出错）：
1. 先只做商品列表页 HTML
2. 确认布局正确
3. 添加 CSS 样式
4. 确认样式正确
5. 添加 JS 交互
```

## 提示词优化技巧

### 技巧 1：明确输出格式

```
❌ 一般：帮我写一封邮件
✅ 明确：帮我写一封求职邮件，格式：
   - 标题：（职位）-（姓名）-（日期）
   - 正文：3段，第一段自我介绍，第二段经历，第三段结尾
   - 字数：200字以内
```

### 技巧 2：提供上下文

```
❌ 缺乏上下文：这段代码有什么问题？
✅ 提供上下文：这是一个 Python 函数，用于计算用户平均消费。
   运行时报错：IndexError: list index out of range
   代码如下...（附代码）
```

### 技巧 3：指定角色

```
❌ 没指定：怎么让用户留存？
✅ 指定角色：作为一个有 10 年经验的产品经理，如何提升 App 用户留存？
```

### 技巧 4：分步骤引导

```
❌ 一步到位：教我 Python 带我做项目
✅ 分步骤：
   第一步：Python 基础语法
   第二步：做一个小计算器
   第三步：接入 API
   第四步：做完整项目
```

### 技巧 5：使用示例

```
❌ 模糊：让回复更友好
✅ 具体示例：
   好的！[具体友好回复示例]
   像这样，语气轻松活泼，多用表情
```

## 上下文管理经验

### 上下文窗口是什么？

Claude Code 有记忆限制，就像人的短期记忆一样。超过限制会"遗忘"早期的内容。

```
上下文窗口示例（简化）：
├── 系统提示 (1000 tokens)
├── 你的问题 (500 tokens)
├── AI 回答 (800 tokens)
├── 你的追问 (500 tokens)
├── AI 回答 (1000 tokens)
└── ... 达到上限后，早期内容会被压缩或遗忘
```

### 管理策略

| 策略 | 适用场景 | 方法 |
|------|----------|------|
| 及时确认 | 每步完成后确认 | "上面这个功能完成了吗？" |
| 定期总结 | 长任务中间 | "目前为止做了什么？" |
| 拆分对话 | 任务相关但复杂 | 开新对话处理子任务 |
| 清理历史 | 发现遗忘时 | /clear 或开新对话 |

### 实际技巧

#### 技巧 1：使用项目记忆

```
在 CLAUDE.md 中写：
当前项目：用户管理系统
后端：Python + FastAPI
前端：Vue.js
数据库：PostgreSQL
目标用户：企业内部员工
```

#### 技巧 2：定期确认进度

```
长任务中间问：
"目前完成了哪些部分？还有什么需要做的？"
```

#### 技巧 3：善用 /compact

```
当对话变长时，使用 /compact 命令压缩上下文
保留核心信息，删除冗余
```

## 调试和修复错误技巧

### 技巧 1：提供完整错误信息

```
❌ 不完整：代码报错了
✅ 完整：
   错误类型：TypeError
   错误信息：'NoneType' object is not iterable
   出错位置：main.py 第 23 行
   触发操作：运行 python main.py
```

### 技巧 2：逐步排查

```
排查思路：
1. 先让 AI 解释为什么可能出错
2. 让 AI 提出修改方案
3. 每次只改一个地方
4. 改完立即测试
5. 确认后再改下一个
```

### 技巧 3：提供最小复现

```
❌ 一大段代码全扔过去
✅ 最小复现：
   "这个报错可以在这个最小函数中复现：
   def test():
       data = None
       return [x for x in data]  # 第 3 行报错
   "
```

### 技巧 4：让 AI 先解释

```
"在修改之前，先解释一下这个错误可能的原因。
然后提出 2-3 个可能的修复方案，我选择后再修改。"
```

## 高效工作流模板

### 模板 1：新项目启动

```
1. [ ] 描述项目目标（1-2 句话）
2. [ ] 说明技术栈约束
3. [ ] 让 AI 生成项目结构
4. [ ] 确认结构后逐步实现
```

### 模板 2：代码审查

```
1. 提供代码范围："审查 src/auth.py"
2. 指定关注点："重点关注安全性"
3. 要求格式："列出问题清单，标出严重程度"
```

### 模板 3：Bug 修复

```
1. 描述预期行为
2. 描述实际行为
3. 提供完整错误信息
4. 提供最小复现代码
5. 等待 AI 分析原因
6. 确认方案后修改
```
