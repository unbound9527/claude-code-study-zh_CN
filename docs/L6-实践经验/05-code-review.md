# 代码审查

## 审查角度

| 角度 | 检查点 |
|------|--------|
| **正确性** | 逻辑是否正确、边界条件处理 |
| **安全性** | SQL注入、XSS、密码明文等 |
| **性能** | 时间/空间复杂度、数据库查询 |
| **可读性** | 命名规范、注释、函数长度 |
| **最佳实践** | 设计模式、错误处理、资源释放 |

## 常用句式

```
请审查 [文件/函数]，重点关注：
1. [角度1]
2. [角度2]
3. [角度3]

输出格式：
- 问题列表（严重程度：高/中/低）
- 改进建议
- 示例代码
```

## 审查流程

```
1. 提供代码范围
   "审查 src/auth/login.py"

2. 指定关注点
   "重点关注安全性"

3. 获取问题清单
   AI 列出发现的问题

4. 按优先级修复
   高 → 中 → 低
```

## 安全审查要点

| 高危 | 说明 |
|------|------|
| SQL拼接 | 必须用参数化查询 |
| 用户输入 | 必须验证和过滤 |
| 密码存储 | 必须加密，不能明文 |
| 权限检查 | 关键操作必须有权限验证 |

## 代码审查模板

```
审查范围：[文件/函数]
重点关注：[安全性/性能/可读性/...]
输出格式：
1. 问题列表
   - [严重] 问题描述 → 建议修复
   - [中等] ...
2. 改进建议
3. 总结
```
