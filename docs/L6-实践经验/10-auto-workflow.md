# 自动化工作流

## 识别可自动化的任务

| 特征 | 说明 |
|------|------|
| 重复性高 | 定期做同样的事 |
| 规则明确 | 有固定的步骤/格式 |
| 耗时多 | 手动做花时间 |

## 常用自动化场景

| 场景 | 输入 | 输出 |
|------|------|------|
| 日/周报生成 | 工作内容 | 格式化报告 |
| 代码审查 | PR 链接 | 审查报告 |
| 会议纪要 | 录音/文字 | 结构化记录 |
| 数据报表 | 原始数据 | 分析报告 |

## 日报自动生成

```
请根据以下信息生成日报：

完成事项：
1. [任务1]
2. [任务2]

进行中：
- [任务3]

下周计划：
- [任务4]

格式：
## 今日完成
## 进行中
## 明日计划
## 备注
```

## PR 审查工作流

```
1. 提供 PR 链接或代码变更
2. 让 Claude Code 审查
3. 生成审查报告
4. 自动评论到 PR
```

## 批量处理

```
❌ 低效：每天单独问
✅ 高效：周五一次性生成下周 5 天日报模板
```

## 工作流设计原则

| 原则 | 说明 |
|------|------|
| 单一职责 | 每个脚本只做一件事 |
| 输入输出明确 | 格式固定 |
| 易于维护 | 加注释、模块化 |
| 可验证 | 能确认执行结果 |
