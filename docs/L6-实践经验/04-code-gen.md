# 代码生成

## 核心原则

| 原则 | 说明 |
|------|------|
| 说清楚输入输出 | 具体的数据格式、返回值 |
| 指定技术栈 | 语言、框架、版本 |
| 提供约束 | 性能、安全、兼容性要求 |

## 常用句式

```
用[语言]写[功能]：
- 输入：[具体说明]
- 输出：[具体说明]
- 约束：[性能/安全/其他要求]
```

## 代码生成场景

### 1. 完整功能

```
"用 Python + FastAPI 实现用户注册 API：
- 输入：邮箱、密码、确认密码
- 输出：user_id、token
- 要求：密码加密存储，邮箱格式验证"
```

### 2. 单个函数

```
"用 JavaScript 写一个防抖函数 debounce：
- 输入：函数、延迟毫秒
- 输出：包装后的函数
- 包含 TypeScript 类型注解"
```

### 3. 代码补全

```
"完成以下函数：
function sum(arr) {
  // 请补全：返回数组所有元素的和
}"
```

### 4. 模板生成

```
"帮我生成一个 React 组件模板：
- 包含 props 类型定义
- 包含 useState 示例
- 包含 PropTypes 验证"
```

## 生成后处理

| 步骤 | 操作 |
|------|------|
| 1 | 先让 AI 解释代码逻辑 |
| 2 | 检查是否符合预期 |
| 3 | 在测试环境运行 |
| 4 | 确认无误再正式使用 |

## 常见问题

| 问题 | 解决 |
|------|------|
| 代码有 bug | 让 AI 解释后重新生成 |
| 不符合项目风格 | 提供代码风格示例 |
| 缺少边界检查 | 明确要求"包含完整错误处理" |
