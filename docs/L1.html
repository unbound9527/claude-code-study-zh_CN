02

清晰表达需求

需求五要素

  • 背景 — 项目、技术栈、架构
  • 任务 — 明确动作+对象+范围
  • 功能细节 — 输入+输出+流程
  • 约束 — 必须/禁止/边界
  • 验收 — 如何算完成

五大误区

  • ❌ 需求模糊，信息缺失
  • ❌ 一次多任务
  • ❌ 不写约束，放任自由
  • ❌ 否定句低效
  • ❌ 不验证，生成即结束

公式：清晰需求 = 背景 + 任务 + 功能 + 约束 + 验收

03

工具使用

三大核心能力

  • 文件系统 — 读取、编辑、创建、删除
  • 命令执行 — 运行测试、构建项目
  • 外部集成 — MCP 连接数据库/API

内置命令

  • /clear — 清空上下文
  • /compact — 智能压缩
  • /context — 查看状态
  • /diff — 查看变更
  • /plan — 仅规划

核心价值：让 AI 从"建议者"变为能落地执行的"开发队友"

04

Git 协作

核心口诀

分支隔离不碰主，原子提交留痕迹，规范提交好回溯，安全协作不踩坑

分支规范

  • feat/ai-collab-功能名 — 新功能分支
  • bugfix/问题模块-描述 — 修复分支
  • 禁止直接操作 main 分支

提交规范

  • 遵循 Conventional Commits
  • 必须标注 (via Claude Code)
  • 原子提交（一个提交只做一件事）

Skills 技能系统

什么是 Skill？

Skill = 标准化工作流 + 可复用提示词 + 自动触发规则

  • SKILL.md 是核心载体
  • 两种触发：手动 /命令 + 自然语言
  • 全模型支持，客户端本地功能

核心价值

  • ✅ 告别重复提示
  • ✅ 自动专业能力
  • ✅ 标准化协作
06

MCP 扩展机制

通俗理解

AI 的通用 USB 接口
MCP 让 Claude 即插即用连接数百种外部工具

两种传输模式

  • Stdio — 本地进程，低延迟
  • HTTP — 远程服务，跨设备

必装 5 个 MCP

  • fs — 文件系统
  • github — 代码协作
  • postgres — 数据库
  • context7 — 最新文档
  • figma — 设计转代码

全模型支持 · 三大能力：Tools（执行）/ Resources（读取）/ Prompts（流程）
