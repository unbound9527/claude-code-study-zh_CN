# 常见工作流

## 本章目标

- 了解 Claude Code 的常用工作流
- 能将工作流应用到日常任务
- 能设计自己的自动化工作流

---

## 什么是工作流？

**工作流 = 完成特定任务的标准步骤**

就像做饭有流程：
1. 准备食材
2. 切菜
3. 炒菜
4. 装盘

Claude Code 也有常用的工作流程。

---

## 工作流一：代码审查

### 场景：检查代码质量

```
步骤 1：让 AI 阅读代码
"请看一下 src/utils.js 的代码"

步骤 2：请求审查
"请从以下几个角度审查：
1. 代码质量
2. 潜在 bug
3. 安全问题
4. 性能考虑"

步骤 3：获取改进建议
"请给出具体的修改建议和示例代码"
```

### 工作流图

```
阅读代码 → 分析问题 → 提供建议 → 实施修改
```

---

## 工作流二：Bug 修复

### 场景：修复程序错误

```
步骤 1：描述问题
"我的程序报错了：
错误信息：[粘贴错误]
发生情况：[描述]

步骤 2：分析原因
"请帮我定位问题的根本原因"

步骤 3：获取修复方案
"请提供修复代码并解释原因"

步骤 4：验证
"请解释修复的逻辑"
```

### 工作流图

```
描述错误 → 定位原因 → 修复代码 → 验证修复
```

---

## 工作流三：学习新技能

### 场景：学习一门新技术

```
步骤 1：概述了解
"请用 5 分钟的话解释 [技术名称] 的核心概念"

步骤 2：深入学习
"请详细解释 [具体概念]，并给出代码示例"

步骤 3：动手练习
"请出一个练习题，让我动手实现"

步骤 4：答疑解惑
"关于 [刚才学的]，我有个问题：[问题]"
```

### 工作流图

```
概览 → 深入 → 实践 → 答疑 → 总结
```

---

## 工作流四：写作修改

### 场景：写一篇文章

```
步骤 1：写初稿
"请帮我写一篇关于 [主题] 的文章，约 500 字"

步骤 2：修改润色
"请修改这篇文章，让语言更生动"

步骤 3：调整结构
"请调整段落结构，让逻辑更清晰"

步骤 4：最终检查
"请检查语法、错别字、标点"
```

### 工作流图

```
初稿 → 润色 → 结构 → 校对 → 完成
```

---

## 工作流五：项目开发

### 场景：从零开始一个项目

```
步骤 1：需求分析
"请帮我分析这个需求：
[粘贴需求]

步骤 2：设计架构
"请设计项目的技术方案"

步骤 3：编写代码
"请按照方案实现核心功能"

步骤 4：测试验证
"请帮我写测试用例并运行"

步骤 5：完善文档
"请为代码添加注释和使用说明"
```

### 工作流图

```
需求 → 架构 → 实现 → 测试 → 文档
```

---

## 工作流六：学习备考

### 场景：复习考试

```
步骤 1：整理知识点
"请把以下章节整理成复习资料：
[粘贴内容]

步骤 2：生成练习
"请根据这些知识点出 10 道选择题"

步骤 3：练习答题
"请检查我的答案：[粘贴答案]"

步骤 4：查漏补缺
"我答错的题目：[列出]
请解释正确思路"
```

### 工作流图

```
整理 → 出题 → 练习 → 批改 → 补缺
```

## 设计自己的工作流

### 步骤一：识别重复步骤

问自己：
- 这个任务有哪些固定步骤？
- 哪些步骤每次都做？
- 哪些步骤可以模板化？

### 步骤二：提取固定模式

```
写作工作流：
1. 写初稿（可变）
2. 润色（固定步骤）
3. 校对（固定步骤）
```

### 步骤三：创建模板

把工作流保存为模板：

```
【我的写作工作流】
1. 请用 [风格] 写 [主题]，约 [字数] 字
2. 请从 [角度] 修改
3. 请检查 [方面]
4. 请调整 [格式]
```

## 常用工作流模板

### 代码审查模板

```
请按以下步骤审查 [文件]：

1. 阅读并理解代码
2. 检查潜在 bug
3. 检查安全问题
4. 检查性能问题
5. 给出改进建议
```

### Bug 修复模板

```
我的程序出现问题：
- 错误：[错误信息]
- 预期：[期望行为]
- 实际：[实际行为]

请帮我：
1. 定位原因
2. 提供修复代码
3. 解释修复逻辑
```

### 学习复习模板

```
请帮我复习 [主题]：

1. 提取核心知识点（5 个）
2. 出 3 道选择题
3. 出 1 道简答题
4. 给出一道综合应用题
```

## 本章小结

**L3 熟练运用**教会你如何高效、巧妙地使用 Claude Code：

| 核心能力 | 关键要点 |
|---------|---------|
| 任务分解 | 大任务拆小任务，降低复杂度，提高完成质量 |
| 上下文管理 | 理解 Context 限制，善用压缩和摘要 |
| 工作流模板 | 建立固定流程模板，重复任务一键完成 |
| Claude.md | 为项目配置专属指令，让 AI 理解你的项目背景 |

**工作流模板汇总**：
```
代码审查：阅读 → 分析 → 建议 → 实施
Bug 修复：描述 → 定位 → 修复 → 验证
学习新技能：概览 → 深入 → 实践 → 答疑
写作修改：初稿 → 润色 → 结构 → 校对
```

**本章重点记住**：复杂任务分解做，简单任务模板化。Claude.md 是你的"项目说明书"。
