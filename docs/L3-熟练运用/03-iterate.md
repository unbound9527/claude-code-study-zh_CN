# 迭代优化答案

## 什么是迭代

```
第一版 → 不满意 → 调整 → 第二版 → 还不够好 → 再调整 → ... → 满意
```

好结果往往需要多轮调整。

## 常见模式

**逐步完善**：方案 → 第三点展开 → 加时间表 → 检查预算

**风格调整**：写文章 → 太正式 → 轻松点 → 加幽默 → 精简

**重构改进**：代码 → 太复杂 → 简化 → 加注释 → 提效

## 调整指令

| 目的 | 说法 |
|------|------|
| 更简洁 | "精简到一半""一句话总结" |
| 更详细 | "展开说说""给更多例子" |
| 改风格 | "口语化""更专业" |
| 换角度 | "从成本角度考虑" |

## 调整层次

| 层次 | 内容 |
|------|------|
| 词汇 | 换词、换说法 |
| 句子 | 改句子结构 |
| 段落 | 调段落顺序 |
| 结构 | 改整体框架 |

## 有效 vs 无效迭代

| ❌ 无效 | ✅ 有效 |
|--------|--------|
| "再写一遍" | "第三段精简" |
| "重新回答" | "换大学生场景的例子" |
| "不对" | "按背景-问题-方案重组" |
