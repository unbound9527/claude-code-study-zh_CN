# Claude.md 的作用

## 本章目标

- 理解什么是 Claude.md
- 知道如何创建和使用 Claude.md
- 能为自己的项目配置 Claude.md

---

## 什么是 Claude.md？

**Claude.md = 项目说明书**

当你打开 Claude Code 在某个目录时，它会自动查找这个目录下的 `CLAUDE.md` 文件，并读取其中的内容来了解项目的背景、规则和上下文。

```
项目目录/
├── CLAUDE.md        ← 这个文件！
├── src/
│   └── ...
└── README.md
```

---

## Claude.md 有什么用？

| 作用 | 说明 |
|------|------|
| **提供背景** | 告诉 AI 关于项目的所有信息 |
| **设置规则** | 定义项目的编码规范、流程 |
| **减少重复** | 不用每次对话都解释项目 |
| **保持一致** | 确保每次回答都符合项目要求 |

---

## Claude.md vs 对话中说明

| 方式 | Claude.md | 对话中说明 |
|------|-----------|------------|
| 持久性 | 永久有效 | 每次对话都要说 |
| 作用范围 | 当前项目目录 | 仅当前对话 |
| 适用场景 | 项目规则、背景 | 临时任务 |

## 如何创建 Claude.md

### 基本结构

```markdown
# 项目名称

这是一个 [描述] 项目。

## 项目背景
[项目是做什么的，为什么存在]

## 技术栈
- 前端：[技术]
- 后端：[技术]
- 数据库：[技术]

## 代码规范
[编码规范、命名规则等]

## 工作流程
[开发流程、提交流程等]

## 注意事项
[需要注意的特殊事项]
```

## 实际示例

### 示例一：前端项目

```markdown
# 我的博客项目

这是一个使用 Next.js 构建的个人技术博客。

## 技术栈
- 框架：Next.js 14
- 样式：Tailwind CSS
- 内容：MDX

## 代码规范
- 组件放在 `components/` 目录
- 页面放在 `app/` 目录
- 使用中文注释

## 工作流程
1. 创建分支
2. 开发功能
3. 提交 PR
4. 合并后自动部署

## 常用命令
- `npm run dev` - 启动开发服务器
- `npm run build` - 构建生产版本
```

### 示例二：学习项目

```markdown
# Python 学习笔记

这是我学习 Python 的项目，记录学习和练习代码。

## 学习目标
- 掌握 Python 基础
- 学会数据分析
- 能独立完成小项目

## 代码规范
- 每个练习单独一个文件
- 文件名格式：`lesson_序号_主题.py`
- 必须有注释解释

## 练习要求
- 完成后再看答案
- 遇到问题先自己思考
```

## Claude.md 的最佳实践

### 1. 放在项目根目录

```
项目根目录/
└── CLAUDE.md   ← 必须在这里
```

### 2. 定期更新

项目变化时更新 CLAUDE.md：
- 添加新的技术栈
- 更新工作流程
- 记录新的规范

### 3. 保持简洁

不要写太长，重点突出：
- 一眼能看完
- 包含最重要的信息
- 便于快速查阅

---

## Claude.md 和 .claude 目录

| 文件/目录 | 作用 |
|-----------|------|
| `CLAUDE.md` | 项目级说明，当前项目有效 |
| `~/.claude/settings.json` | 全局设置，所有项目有效 |
| `~/.claude/skills/` | 全局技能，所有项目有效 |

---

## 跟做

1. 进入一个你已有的项目目录
2. 创建 `CLAUDE.md` 文件
3. 填写项目的基本信息
