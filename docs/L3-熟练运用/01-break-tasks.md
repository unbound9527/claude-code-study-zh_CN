# 分解复杂任务

## 本章目标

- 理解为什么复杂任务需要分解
- 学会将大任务拆成小步骤
- 能用"分步骤"方式处理复杂问题

---

## 为什么要分解任务？

### 大任务 vs 小任务

| 大任务（难） | 小任务（易） |
|--------------|--------------|
| "帮我写一本书" | "帮我写第一章" |
| "帮我装修房子" | "帮我设计客厅布局" |
| "帮我准备考试" | "帮我整理第三章知识点" |

**大任务 = 多个小任务的组合**

---

## 分解任务的思维

### 方法：金字塔思维

```
顶层：最终目标
中层：主要步骤（3-5 个）
底层：具体行动
```

### 示例：准备演讲

```
最终目标：准备一场 10 分钟的演讲
│
├── 第一步：确定主题和结构
│   ├── 确定核心观点（1 个）
│   ├── 列出 3 个分论点
│   └── 设计开头和结尾
│
├── 第二步：撰写内容
│   ├── 写开场白
│   ├── 写正文（3 段）
│   └── 写结尾
│
└── 第三步：准备 Q&A
    ├── 预测 5 个问题
    └── 准备答案
```

## 分解任务的信号

### 需要分解的情况

| 信号 | 说明 |
|------|------|
| 问题太宽泛 | "怎么成功？" |
| 涉及多个领域 | 需要不同专业知识 |
| 步骤不明确 | 不知道从哪里开始 |
| 预期结果不具体 | "做好一点" |

## 和 Claude Code 分解任务

### 错误方式

```
❌ 一步到位：
"帮我从零开始做一个小程序"

问题：太复杂，AI 不知道从何说起
```

### 正确方式

```
✅ 分步骤：
"我要做一个待办事项小程序，请按这个顺序帮我：

第一步：告诉我做这个小程序需要哪些步骤
第二步：帮我设计数据库结构
第三步：帮我写前端页面
..."

或者直接说：
"我要做一个待办事项小程序。请先告诉我需要哪些步骤，一步一步来。"
```

## 子任务模板

```
我要做 [最终目标]，请按以下步骤帮我：

第一步：[子任务 1]
第二步：[子任务 2]
第三步：[子任务 3]

每完成一步请等我确认后再进行下一步。
```

## 实战练习

### 练习：准备论文

**任务：** 写一篇 5000 字的毕业论文

**分解方案：**

```
第一步：确定论文框架
- 确定研究问题
- 列出章节大纲
- 确认研究方法

第二步：撰写各章节
- 摘要（300 字）
- 引言（500 字）
- 文献综述（1500 字）
- 研究方法（800 字）
- 结果分析（1500 字）
- 结论（400 字）

第三步：整理参考文献
```

## 跟着做

1. 想一个你想完成的大任务（比如学一门新技能、做一个项目）
2. 打开 Claude Code
3. 输入：

```
我想 [你的大任务]，请帮我分解成 5 个具体的步骤。
```

4. 根据 AI 的建议，再细化每个步骤

- A. 具体行动
- B. 最终目标

- A. "帮我写第一章"
- B. "帮我写一本书"
