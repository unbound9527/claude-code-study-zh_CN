# 管理对话上下文

## 本章目标

- 理解什么是"上下文"
- 知道上下文会有限制
- 学会管理对话长度

---

## 什么是上下文？

**上下文 = AI 能记住的对话内容**

就像人的记忆一样，AI 也有记忆容量限制：
- 对话太长，AI 可能会忘记之前的部分
- AI 会优先记住最近说的话

---

## 上下文限制的表现

当上下文接近满时，你可能会发现：

| 现象 | 原因 |
|------|------|
| AI 开始"忘记"你说的 | 对话太长 |
| 回答质量下降 | 核心信息被稀释 |
| AI 建议你"压缩" | 需要清理上下文 |

---

## 管理上下文的技巧

### 技巧一：简洁提问

❌ 啰嗦：
```
之前我们讨论了 A，然后 B 也说过了，接着 C 又提到了，然后我想问你...
```

✅ 简洁：
```
继续说 C 的内容
```

### 技巧二：开启新对话

当话题转换时：

```
对话 1：论文第一章 → 完成 → 关
对话 2：论文第二章 → 开始 → 开
```

### 技巧三：定期总结

对于长期项目，让 AI 定期总结：

```
"请先总结一下我们之前讨论的要点，然后继续"
```

## 使用 /compact 命令

当对话太长时，输入：

```
/compact
```

**作用：** 压缩对话历史，保留关键信息，让 AI 轻装上阵。

### compact 前 vs compact 后

```
compact 前：
- 100 条消息
- AI 记忆了很多细节

compact 后：
- 保留核心要点
- AI 重新轻装
```

## 三层 Context Compact

Claude Code 提供三个层级的上下文压缩，帮你更精细地管理对话：

### 第一层：轻度压缩（/compact）

**什么时候用**：对话刚开始变长，但核心内容还有用

**效果**：
- 保留所有关键结论
- 压缩重复的确认对话
- 保留最近的 5-10 轮完整对话

**触发方式**：
```
/compact
```

### 第二层：中度压缩（/compact full）

**什么时候用**：对话很长，需要更彻底地清理

**效果**：
- 只保留关键决策点
- 压缩中间过程
- 保留项目目标、约束条件、最终方案

**触发方式**：
```
/compact full
```

### 第三层：深度压缩（/compact summary）

**什么时候用**：对话已经严重膨胀，但还想继续

**效果**：
- 只保留高层摘要
- 所有细节转为"根据之前的讨论..."
- 最大程度节省上下文空间

**触发方式**：
```
/compact summary
```

---

### 三层压缩对比

| 层级 | 触发命令 | 保留内容 | 适用场景 |
|------|----------|----------|----------|
| 轻度 | `/compact` | 关键结论 + 最近对话 | 对话变长时定期清理 |
| 中度 | `/compact full` | 决策点 + 目标 | 需要更彻底清理时 |
| 深度 | `/compact summary` | 高层摘要 | 上下文严重不足时 |

---

### 使用建议

```
日常使用：轻度压缩（/compact）
    ↓ 对话继续变长
中度压缩（/compact full）
    ↓ 还是不够
深度压缩（/compact summary）
    ↓ 真的不够了
开启新对话 + 总结背景
```

---

## 上下文使用指南

| 对话长度 | 建议 |
|----------|------|
| 短（< 10 轮） | 直接继续 |
| 中（10-20 轮） | 注意简洁 |
| 长（> 20 轮） | /compact 或开新对话 |

---

## 什么时候开新对话？

| 情况 | 建议 |
|------|------|
| 话题转换 | 开新对话 |
| 任务完成 | 开新对话 |
| AI 开始"失忆" | /compact 或开新对话 |
| 项目太大 | 按模块分开对话 |

## 跟做

1. 打开一个对话，问 5 个不同的问题
2. 问第 6 个问题时，先说："请总结一下我之前问了什么"
3. 观察 AI 记得多少

---

## 练一练

1. 开启一个长对话，问 AI 帮你写一个故事的不同部分
2. 写到第 5 部分时，用 /compact
3. 然后问："故事到现在讲了什么？"
4. 观察内容是否连贯
