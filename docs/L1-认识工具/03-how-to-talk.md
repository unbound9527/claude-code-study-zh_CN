# 怎么和它说话

## 本章目标

- 理解和 AI 对话的基本格式
- 学会正确地提出问题
- 避免常见的对话误区

---

## 基本对话格式

Claude Code 的对话很简单：

```
你：输入你的问题或需求
AI：回复你
你：追问或新问题
AI：继续回复
...
```

就像和朋友发微信一样！

---

## 好问题和坏问题

### 坏问题 vs 好问题

| 坏问题 | 好问题 | 为什么 |
|--------|--------|--------|
| "帮我看看" | "帮我检查这段文字的语法" | 太模糊 |
| "怎么做" | "我想做一个番茄钟 APP，请告诉我步骤" | 缺少背景 |
| "那个事" | "上周你帮我写的那封邮件，能再发一遍吗" | 指代不清 |

---

## 提问的黄金法则

### 法则一：说清楚你要什么

❌ 不清楚：
```
帮我写东西
```

✅ 清楚：
```
帮我在 100 字以内写一句朋友圈文案，推荐我新买的耳机
```

### 法则二：给出必要的背景

❌ 没背景：
```
这个方案怎么样？
```

✅ 有背景：
```
我是一个大学生，想在校园卖零食，这个定价方案合理吗：
每包薯片定价 8 元，进价 5 元...
```

### 法则三：指定格式

❌ 没要求：
```
给我列几个学习方法
```

✅ 有要求：
```
给我 5 个高效学习方法，用表格形式，包含：方法名、具体步骤、适用场景

## 像聊天但不是聊天

### 相似之处

- 可以像朋友一样说话
- 可以追问
- 可以让它调整

### 不同之处

| 聊天 | Claude Code |
|------|-------------|
| 朋友会记得上次说的 | 除非开启记忆功能，否则不一定记得 |
| 朋友会猜你的意思 | AI 只能按你说的理解 |
| 朋友会纠正你 | AI 会尽量回答你 |

## 多窗口管理

有时候你可能同时处理多个项目或话题，需要管理多个对话窗口。

### 为什么要多窗口？

| 场景 | 例子 |
|------|------|
| 不同项目 | 一个窗口写论文，一个窗口写代码 |
| 不同话题 | 一个问学习问题，一个问生活问题 |
| 整理资料 | 一个窗口整理笔记，一个窗口出练习题 |

### 创建新窗口

1. 点击左侧边栏的 **"新建对话"** 或 **"New Chat"**
2. 输入新话题
3. 开始新对话

### 管理对话列表

| 操作 | 方法 |
|------|------|
| 查看历史 | 点击左侧边栏的对话列表 |
| 搜索对话 | 在搜索框输入关键词 |
| 删除对话 | 鼠标悬停 → 点击删除按钮 |
| 重命名对话 | 鼠标悬停 → 点击编辑按钮 |

### 组织技巧

给对话起清晰的名字，方便以后查找：

```
❌ 不清楚：对话 1
✅ 清楚：英语学习-语法问题
✅ 清楚：论文第二章修改
✅ 清楚：Python 作业答疑
```

## 跟着做

1. 创建一个新对话
2. 给对话起名：`测试对话`
3. 输入：`请用 3 句话介绍你自己`
4. 关闭这个对话
5. 在对话列表中找到它并重新打开

---

## 练一练

1. 创建一个新对话
2. 问一个具体的问题（参考"好问题"格式）
3. 根据回答追问
4. 给对话起一个有意义的名字

## 小测验

**1. 给 AI 提问时，最重要的是什么？**
- A. 用英文提问
- B. 说清楚你要什么
- C. 越短越好

**2. Claude Code 默认记得之前的对话吗？**
- A. 永远记得
- B. 记得所有对话
- C. 只记得当前对话，除非开启记忆

**3. 给对话起名字的目的是？**
- A. 没有意义
- B. 方便以后查找
- C. 让 AI 更聪明

---

## 答案

1-B | 2-C | 3-B
