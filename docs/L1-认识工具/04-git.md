# 什么是 Git？

## 本章目标

- 理解什么是 Git
- 知道 Git 和 GitHub 的区别

---

## Git 是什么？

**Git = 版本控制系统**

想象你写论文时的场景：
- 论文_v1.docx
- 论文_v2.docx
- 论文_最终版.docx
- 论文_真的最终版.docx
- 论文_老师看了终于过了.docx

**Git 就是帮你管理这些版本的工具**，让你不再需要一堆乱七八糟的文件名。

---

## 用比喻理解 Git

| 概念 | 比喻 |
|------|------|
| **Git** | 时光机，记录每次修改 |
| **仓库（Repository）** | 项目的存档柜 |
| **提交（Commit）** | 一次存档，保存当前状态 |
| **分支（Branch）** | 从主线分叉出新故事线 |
| **合并（Merge）** | 把分叉的故事合并在一起 |

---

## Git 和 GitHub 是一回事吗？

**不是！** 它们的关系：

| | 是什么 | 比喻 |
|--|--------|------|
| **Git** | 版本控制的工具/软件 | 相机（拍摄快照） |
| **GitHub** | 存放 Git 仓库的网站 | 云相册（存储照片） |

```
Git = 工具 → 本地电脑上
GitHub = 服务 → 云端
```

---

## Git 有什么用？

| 场景 | 没有 Git | 有 Git |
|------|----------|--------|
| 查看修改历史 | 翻找一堆文件 | 每次提交都有记录 |
| 多人协作 | 来回传文件 | 各自开发，最后合并 |
| 出错回退 | 手动还原 | 一键回到之前的版本 |
| 备份 | 只有本地一份 | 云端也有一份 |

## Git 基本概念

### 仓库（Repository）

一个 Git 仓库 = 一个项目

```
my-project/           ← 这是一个仓库
├── .git/            ← 隐藏文件夹，Git 在这里记录一切
├── 文件1
└── 文件2
```

### 提交（Commit）

```
提交 = 拍一张快照

commit "添加用户登录功能"
├── 修改了 login.js
├── 修改了 auth.css
└── 新增了 user.js
```

每次提交都有：
- **描述**：这次改了什么
- **时间**：什么时候改的
- **作者**：谁改的

### 分支（Branch）

```
main        ────●────●────●
                ↑
              分支 (feature/login)

feature/login  ────●────●
                       ↑
                   合并回 main
```

## 为什么 Claude Code 需要 Git？

Claude Code 可以帮你：

| 操作 | 说明 |
|------|------|
| 查看修改 | "帮我看看这次改了什么" |
| 保存版本 | "帮我提交这次修改" |
| 查看历史 | "帮我看看之前的版本" |
| 创建分支 | "帮我开一个分支" |
| 合并代码 | "帮我把这两个分支合并" |

---

## Git 常用命令

| 命令 | 作用 |
|------|------|
| `git init` | 初始化一个新仓库 |
| `git add .` | 把修改的文件加入暂存区 |
| `git commit -m "信息"` | 提交，保存当前状态 |
| `git status` | 查看当前状态 |
| `git log` | 查看提交历史 |
| `git branch` | 查看分支 |
| `git checkout` | 切换分支 |
| `git merge` | 合并分支 |
| `git push` | 推送到远程仓库 |
| `git pull` | 从远程拉取更新 |

## 跟做

2. 在文件夹里初始化 Git：`git init`
3. 创建一个文件
4. 提交：`git add .` → `git commit -m "第一次提交"`
