04

权限管理

5 种权限模式

  • default — 新手推荐，读自动，写/执行确认
  • acceptEdits — 主力模式，日常首选
  • plan — 仅允许读/搜索/分析
  • auto — 内置安全分类器
  • dontAsk — 全部自动通过（危险）

三层配置

  • User — ~/.claude/settings.json
  • Project — .claude/settings.json
  • Local — .claude/settings.local.json

deny > ask > allow

06

Subagent 子代理

核心概念

独立的专用 AI 助手，拥有独立上下文、自定义系统提示词、细粒度工具权限

两种创建方式

  • /agents — 交互式（推荐）
  • Markdown 配置 — 手动创建

适用场景

  • 代码审查/质量把控
  • 问题调试/故障定位
  • 文档生成/维护
  • 项目结构探索/分析
  • 测试用例编写
07

Agent Teams 多智能体协作

与 Subagent 区别

  • Subagent = 主仆分工
  • Agent Teams = 平级团队协作
  • 完全独立实例，上下文隔离
  • 队友间直接点对点通信

重要限制

仅支持 Claude 3（Opus/Sonnet/Haiku）
不支持国产模型

启用条件

  • Claude Code v2.1.32+
  • Pro / Max 订阅
  • 手动启用：
    CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1

适用：大型项目并行开发、多角度审查

Computer Use

核心能力

让 AI "看见"屏幕、操控鼠标与键盘，突破终端文本交互
