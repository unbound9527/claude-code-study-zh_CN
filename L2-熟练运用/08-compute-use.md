# Claude Code Computer Use 完全教学：让 AI 直接操控你的电脑
**Computer Use（计算机使用）** 是 Claude Code 于 2026 年 3 月推出的革命性能力，让 AI 突破终端文本交互，**像真人一样“看见”屏幕、操控鼠标与键盘**，自主完成 GUI 操作、应用测试、跨软件自动化等任务。它是 Claude 从“代码生成助手”升级为“全流程智能代理”的核心标志。

---

## 一、什么是 Computer Use？
### 1. 核心定义
**Computer Use** 是 Claude Code 内置的 **MCP 服务（computer-use）**，通过 **屏幕录制（视觉感知）+ 辅助功能（键鼠控制）** 形成 **感知 → 理解 → 决策 → 执行 → 验证** 的完整智能闭环。

### 2. 通俗理解
- **AI 的“眼睛和手”**：能看屏幕、点按钮、输文字、滚动、截图、切换窗口
- **无 API 自动化**：不依赖应用接口，**任何可见 GUI 都能操作**（浏览器、Figma、IDE、系统设置）
- **端到端开发闭环**：写代码 → 编译 → 启动 → 点击测试 → 修复 Bug → 验证，**全程无需手动干预**

### 3. 核心能力（26 类内置工具）
- **视觉**：截屏、分析界面、识别元素、验证结果
- **控制**：点击、双击、右键、输入、滚动、拖拽、窗口调整
- **系统**：启动应用、切换窗口、文件管理、系统设置
- **协作**：跨应用操作、批量任务、错误处理、自动重试

### 4. 与传统自动化区别
| 特性 | Computer Use | 脚本自动化（Selenium/Playwright） |
|:--- |:--- |:--- |
| **原理** | 视觉理解 + 自主决策 | 预定义步骤、硬编码选择器 |
| **适应性** | 界面变化自动适配 | 界面改动即失效 |
| **范围** | 全系统（任何应用） | 仅限浏览器/特定应用 |
| **门槛** | 自然语言指令 | 需编写代码、维护脚本 |
| **智能** | 自主处理弹窗、异常、重试 | 严格按步骤执行，无容错 |

## 二、支持条件（必须满足）
### 1. 系统与版本
- **操作系统**：**仅支持 macOS**（Windows/Linux 暂未支持）
- **Claude Code**：**v2.1.85+**（2026 年 3 月后稳定版）
- **订阅计划**：**Claude Pro / Max**（免费版不可用）
- **模式**：**仅交互式会话**（不支持 `-p` 非交互模式）

### 2. 必备权限（首次启用必授）
- **辅助功能（Accessibility）**：允许 Claude 模拟鼠标点击、键盘输入
- **屏幕录制（Screen Recording）**：允许 Claude 截取屏幕、分析界面
> ⚠️ 授权后需 **重启 Claude Code** 才能生效

## 三、启用步骤（3 分钟搞定）
### 方式 1：CLI 终端（推荐）
```bash
# 1. 启动交互式会话
claude

# 2. 打开 MCP 面板
/mcp

# 3. 启用 Computer Use
# 在 MCP 列表找到 → computer-use → 选择 Enable
# 首次启用会弹出系统权限请求，全部允许
```

### 方式 2：桌面版（Claude Cowork）
1. 打开 Claude 桌面 App
2. 进入 `Settings → Desktop App → General`
3. 打开 **Computer Use** 开关
4. 授权系统权限 → 重启 App

### 验证启用成功
- 会话内输入：`/mcp` → 显示 `computer-use: Connected`
- 底部状态栏：`Computer Use: Enabled`

## 四、基本使用：自然语言指令（零代码）
### 1. 核心语法（直接复制）
```bash
# 开发场景（最常用）
# 1. 应用开发全流程
写一个 macOS 菜单栏计时器应用，用 SwiftUI。写完后编译、启动、点击开始/暂停/重置验证功能，截图所有状态。

# 2. UI Bug 修复
修复设置弹窗在窄窗口下被裁剪的问题。复现 Bug → 截图 → 修改 CSS → 重新验证 → 确认修复。

# 3. 自动 E2E 测试（无需 Playwright）
启动我的 React 应用，依次点击首页 → 登录页 → 个人中心 → 设置，检查所有页面无崩溃，截图错误。

# 办公自动化
整理桌面 "销售数据.xlsx"，生成 PPT，保存到桌面，然后打开企业微信发送到团队群。

# 系统操作

# 浏览器操作
```

### 2. 指令编写技巧（提升成功率）
- **明确目标**：`启动应用 → 点击所有标签 → 截图错误`（而非 `测试应用`）
- **指定范围**：`仅修改 src/components/Modal.css`
- **约束步骤**：`先截图 → 再修改 → 最后验证`
- **结果要求**：`所有操作完成后汇总报告，附 3 张关键截图`

## 五、运行表现（你会看到什么）
### 1. 执行过程（实时可见）
- **自动截屏**：屏幕闪烁一下（AI 分析当前界面）
- **键鼠自动操作**：鼠标自主移动、点击；键盘自动输入
- **会话内日志**：
  ```
  [Computer Use] 截屏分析界面
  [Computer Use] 点击按钮 "启动应用"
  [Computer Use] 输入文本 "test@example.com"
  [Computer Use] 滚动页面到底部

  ```
- **多轮循环**：AI 会 **反复截屏 → 验证 → 调整**，直到任务完成

### 2. 权限控制（安全第一）
- **默认确认**：高危操作（如删除文件、输入密码）**必须手动确认**
- **权限面板**：`/permissions` 查看 Computer Use 权限范围
- **紧急停止**：按 `Ctrl+C` 立即终止所有操作

### 3. 输出结果
- **自动报告**：任务完成后生成 **文字总结 + 截图附件**
- **文件产出**：代码、文档、截图自动保存到项目目录
- **错误反馈**：失败时明确说明原因（如 `未找到按钮 "登录"`）

## 七、权限与安全（必看）
### 1. 权限范围（默认最小）
- **允许**：截屏、点击、输入、启动应用、读写项目文件
- **禁止**：**密码输入、系统设置修改、删除关键文件、访问敏感目录**
- **确认**：所有跨目录操作、高危命令 **必须手动确认**

### 2. 安全配置（推荐）
```json
// .claude/settings.json
{
  "permissions": {
    "computerUse": {
      "enabled": true,
      "allowScreenshot": true,
      "allowInput": true,
      "allowFileAccess": ["./src", "./dist"],
      "denyPaths": ["~/.ssh", "~/Documents", ".env"],
      "requireApproval": ["delete", "password", "systemSettings"]
    }
  }
}
```

### 3. 安全最佳实践
- **生产环境禁用**：仅在 **开发/测试机** 使用
- **最小权限**：仅开放项目目录，禁止访问系统/敏感文件夹
- **全程监控**：首次使用 **盯着执行过程**，熟悉后再后台运行
- **紧急停止**：`Ctrl+C` 随时终止
- **日志审计**：`~/.claude/logs/computer-use-*.log` 全操作记录

## 八、常见问题与避坑
### 1. 问题：启用后无反应
- 检查：**权限已全部授权**（辅助功能 + 屏幕录制）
- 检查：**重启 Claude Code**
- 检查：版本 ≥ **v2.1.85**
- 检查：**Pro/Max 订阅**

### 2. 问题：AI 找不到按钮/输入框
- 解决：指令更具体（`点击右上角蓝色 "登录" 按钮`）
- 解决：确保界面 **可见、未遮挡**
- 解决：窗口 **最大化**，避免元素被裁剪

### 3. 问题：操作太快/出错
- 解决：指令加入 `慢一点，每步间隔 1 秒`
- 解决：`出错时自动重试 2 次，仍失败则报告`

### 4. 问题：不支持 Windows
- 官方：**暂仅 macOS**，Windows/Linux 规划中
