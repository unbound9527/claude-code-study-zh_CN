# 掌控对话记忆：Claude Code 上下文管理完全实战指南
使用 Claude Code 开发时，你是否常遇到这些问题：AI 突然忘记之前的约定、代码逻辑前后矛盾、长对话后响应变混乱、跨文件协作时总需重复解释项目背景？**根源只有一个：上下文失控。**

上下文是 Claude Code 的“短期记忆”与“工作内存”，直接决定代码质量、协作效率与开发流畅度。本文从原理、工具、策略到实战，系统讲解如何**主动管理、精准控制、高效复用**对话上下文，让 AI 始终牢记项目核心、不跑偏、不遗忘。

---

## 一、先懂原理：Claude Code 上下文的本质与构成
### 1. 什么是上下文窗口？
**上下文窗口（Context Window）** 是 Claude 能同时“记住”并处理的**最大信息容量**（默认约 **200K tokens**，支持扩展至 **100万 tokens**）。可类比为：**AI 的有限工作台**——所有信息都要放在台面上才能被处理，台面满了就必须清理旧内容。

### 2. 上下文由 5 部分组成（全在消耗 tokens）
- **系统提示（System Prompt）**：内置规则、编码规范、能力约束（静态，可缓存）
- **CLAUDE.md**：项目级持久记忆（手动维护，自动加载）
- **对话历史**：你与 AI 的所有问答、指令、讨论
- **文件内容**：通过 `@文件` 或读取加载的代码/文档
- **工具结果**：命令执行、搜索、代码运行输出

### 3. 核心危机：上下文溢出与污染
- **溢出（Overflow）**：信息超容量 → AI 自动丢弃旧内容 → 记忆断裂、逻辑混乱
- **污染（Pollution）**：混入错误假设、无关讨论、废弃方案 → AI 被误导、输出偏差
- **低效（Inefficiency）**：加载无关文件、重复提问、冗余信息 → tokens 浪费、响应变慢

**上下文管理的核心目标**：**保持窗口精简、干净、聚焦，只保留当前任务必需信息**。

## 二、核心工具：3 大上下文管理命令（必掌握）
### 1. `/clear` —— 彻底重置，清空污染
**作用**：**完全清空对话历史、文件缓存、工具结果**，仅保留 **CLAUDE.md** 与系统提示。
**本质**：**任务切换的“重启键”**，告别脏上下文。

**✅ 最佳使用场景**
- 切换**完全无关**的新任务（如从登录模块 → 支付模块）
- 调试陷入死循环、AI 持续出错、上下文严重污染
- 长对话后（>50 轮），清理冗余，轻装上阵
- 修正 2 次仍错误 → 直接 `/clear` 重来（官方建议）

**❌ 误区**：舍不得用 `/clear`，怕丢记忆 → 带着污染继续，效率更低。

### 2. `/compact` —— 智能压缩，保留精华
**作用**：将长篇对话**蒸馏为结构化摘要**，压缩率 **60%~70%**，释放空间且保留核心。
**本质**：**上下文的“精炼器”**，适合任务未完成但需减负。

**✅ 基础用法**
```bash
/compact  # 自动压缩全对话
```

**✅ 进阶用法（定向压缩，最强技巧）**
```bash
# 保留架构与API，丢弃调试日志
/compact Preserve: 架构决策、API契约、数据库设计；Discard: 调试输出、失败尝试

# 聚焦认证流程
/compact Focus on: 用户认证模块的实现细节

# 保留文件清单与任务状态
/compact Keep: 修改文件列表、当前任务进度、验收标准
```

**✅ 最佳时机**
- 完成里程碑/子任务，进入下一阶段前
- 上下文占用 >**70%**（用 `/context` 查看），AI 开始“走神”
- 长对话（>30 轮），但任务仍需继续
- 探索性讨论结束，保留结论、丢弃过程

### 3. `/context` —— 监控状态，心中有数
**作用**：查看**当前上下文占用率、token 分布、加载文件、历史摘要**。
**本质**：**上下文的“监控仪表盘”**，避免被动溢出。

**示例输出**
```
上下文状态：
- 总容量：200,000 tokens
- 已使用：145,320 (72.7%)
- 对话历史：65,000
- 加载文件：58,320 (3个文件: user.js, auth.js, db.js)
- CLAUDE.md：12,000
- 工具结果：10,000
```

**✅ 必做动作**
- 每 **15~20 轮对话** 查一次 `/context`
- 占用 **>70%** → 主动 `/compact`
- 占用 **>90%** → 立即 `/compact` 或 `/clear`（避免自动压缩失控）

## 三、长效机制：CLAUDE.md —— 项目级“永久记忆”
### 1. 什么是 CLAUDE.md？
项目根目录下的**自动加载记忆文件**，**每次新会话都会自动读取**，无需重复介绍项目。相当于：**AI 的项目说明书 + 团队知识库**。

### 2. 必写内容（标准模板）
```markdown
# 项目：全栈待办应用
## 1. 基础信息
- 技术栈：React 18 + Node.js + MongoDB + TypeScript
- 架构：前后端分离、RESTful API、JWT 认证
- 核心模块：用户认证、待办 CRUD、权限控制

## 2. 编码规范
- 命名：驼峰法、接口前缀 I、类型后缀 Type
- 错误处理：统一返回 { code, msg, data }
- 必须加注释：函数、接口、复杂逻辑
- 禁止：console.log 生产代码、硬编码密钥

## 3. 常用命令
- npm run dev：启动开发服务
- npm test：运行单元测试
- npm run build：构建生产包

## 4. 关键决策（动态更新）
- 2026-04-01：选用 bcrypt 加密密码，禁用 MD5
- 2026-04-01：API 版本化 /api/v1/**
- 2026-04-01：数据库字段统一用下划线命名
```

### 3. 3 个黄金用法
1. **会话初始化**：新项目必建 `CLAUDE.md`，新会话**零成本理解项目**
2. **动态更新**：每完成重要任务，让 AI 同步更新
   > "总结本次对话的关键决策，更新到 CLAUDE.md 的「关键决策」部分"
3. **跨会话复用**：关闭再打开，AI 自动加载，**无需重述背景**

## 四、高级策略：4 大上下文管理心法（从新手到专家）
### 心法 1：单一任务原则 —— 一次只做一件事
**核心**：**一个会话 = 一个独立任务**，不混 unrelated 需求。
**反例**（混乱）：一边改登录 Bug，一边问页面样式，一边讨论数据库优化
**正例**（清晰）：
- 会话 1：修复登录接口 401 错误
- 会话 2：实现注册页 UI
- 会话 3：设计用户表索引

**好处**：上下文极简、无干扰、易管理、易回溯。

### 心法 2：主动压缩 —— 别等溢出再补救
**工作流（推荐）**
1. 新任务 → `/clear`（干净起点）
2. 需求讨论 → Plan Mode 拆解 → **完成后 /compact**（保留计划、丢弃讨论）
3. 分步编码 → 每完成 2~3 步 → `/compact`
4. 任务结束 → 总结更新 `CLAUDE.md` → `/clear` 迎接下一个任务

### 心法 3：文件引用 —— 少粘贴、多链接
**错误**：直接粘贴大段代码/文件到对话 → 极速占满上下文
**正确**：用 `@文件路径` 精确引用
```bash
# 正确：引用3个相关文件，AI 自动加载上下文
@src/models/user.js @src/routes/auth.js @src/middleware/jwt.js

# 实现用户注册接口，遵循 CLAUDE.md 规范
```

**进阶**：复杂问题**同时引用关联文件**（如路由 + 控制器 + 模型 + 类型），AI 能理解完整链路。

### 心法 4：子代理（Subagent）—— 隔离探索，保护主上下文
**核心**：复杂调研、代码审查、大规模分析 → 交给**独立子代理**（独立上下文），结果再汇总到主会话。
**适用**：
- 大规模代码审计、技术方案调研
- 不确定的探索性需求（避免污染主会话）
- 跨模块、跨文件的复杂分析

**用法**
```bash
# 启动子代理审查代码
/subagent 审查 src/controllers 下所有接口的安全漏洞
# 子代理完成后，输出总结，主会话再决策
```

## 五、实战工作流：完整上下文管理闭环（直接套用）
### 场景：开发“用户认证模块”（长任务、多文件、跨会话）
#### Step 1：准备（新会话）
1. 确认 `CLAUDE.md` 已更新（技术栈、规范、决策）
2. 执行 `/clear` → 100% 干净上下文
3. 查看 `/context` → 确认仅加载 `CLAUDE.md`

#### Step 2：规划（Plan Mode）
```bash
我要实现用户认证模块（注册、登录、JWT、密码加密）。
请用 Plan Mode 拆解为里程碑 + 子任务，明确文件路径与依赖。
```
- AI 输出计划 → 确认无误
- 执行 `/compact Preserve: 实施计划、文件清单、验收标准`

#### Step 3：分步执行（每步闭环）
```bash
# 指令模板（单一职责）
【任务】仅实现：用户注册接口（POST /api/v1/auth/register）
【文件】@src/routes/auth.js @src/models/user.js
【要求】遵循 CLAUDE.md 规范，密码 bcrypt 加密，邮箱唯一性校验
```
1. AI 生成代码 → 你验证/测试
2. 确认正确 → `/compact`（保留当前代码与状态）
3. 进入下一步，重复直到模块完成

#### Step 4：收尾（任务结束）
1. 让 AI 总结：**所有决策、代码变更、注意事项**
2. 指令：`更新 CLAUDE.md 的「关键决策」与「模块进度」`
3. 执行 `/clear` → 上下文重置，准备下一个任务

#### Step 5：跨会话恢复（第二天继续）
1. 启动新会话 → AI **自动加载 CLAUDE.md**
2. 查看 Session Memory：`Recalled X memories`（过往摘要）
3. 指令：`回顾我们之前的用户认证模块进度，当前应从哪一步继续？`
4. 确认后直接继续，**无需重述任何背景**

## 六、常见误区与避坑清单
### ❌ 误区 1：对话越长越好 → 错！
- **真相**：长对话 = 高占用 + 高污染 + 低精准
- **正确**：**任务完成即 `/clear`**，小步快跑、定期压缩

### ❌ 误区 2：舍不得 `/clear`，怕丢记忆 → 错！
- **真相**：重要记忆已写入 `CLAUDE.md`，对话历史只是过程
- **正确**：`/clear` 是**保护机制**，切断污染、重置状态

### ❌ 误区 3：一次性加载所有文件 → 错！
- **真相**：无关文件占满上下文，AI 无法聚焦
- **正确**：**按需引用**，一次只加载当前任务必需文件

### ❌ 误区 4：依赖自动压缩 → 错！
- **真相**：自动压缩是**兜底**，常无差别丢弃，可能丢失关键信息
- **正确**：**主动 `/compact` + 定向保留**，完全可控

## 七、总结：上下文管理 5 字诀
**清**：任务切换必 `/clear`，保持起点干净
**缩**：占用超 70% 必 `/compact`，主动减负
**存**：关键决策写入 `CLAUDE.md`，永久记忆
**精**：一次一任务、按需引文件、拒绝冗余
**查**：常看 `/context`，监控状态、防溢出

**上下文管理不是技巧，是与 Claude Code 协作的底层能力。** 管好它，AI 就是精准高效的资深队友；管不好，AI 就是健忘混乱的新手。从今天起，用这套体系，让你的每一次对话都**记忆清晰、逻辑连贯、产出稳定**。
