# Claude Code Agent Teams 完全教学：多AI协作开发、模型要求与运行表现
Agent Teams 是 Claude Code 2026年初推出的**实验级多智能体协作功能**，可让你一键组建AI开发团队，多个独立AI实例并行工作、自主沟通、协同完成复杂项目。本文完整讲解启用、创建、使用、模型要求与实际表现，帮你掌握“一人顶一个开发团队”的能力。

---

## 一、核心概念：Agent Teams 是什么（与 Subagent 区别）
### 1. 基本定义
- **Team Lead（领队）**：你的主 Claude 会话，负责统筹、拆任务、分配角色、合成结果、管控进度
- **Teammates（队友）**：**完全独立的 Claude Code 实例**（非子会话），有专属上下文、独立权限、独立模型
- **共享任务列表**：统一看板（待处理/进行中/已完成），支持任务依赖
- **内部邮箱（Mailbox）**：队友间**直接点对点通信**，无需经过领队转发

### 2. 与 Subagent（子代理）的关键区别
| 特性 | Subagent（子代理） | Agent Teams（智能体团队） |
| :--- | :--- | :--- |
| **独立性** | 依附主会话，上下文共享/汇总 | 完全独立实例，上下文彻底隔离 |
| **通信方式** | 仅向主代理汇报 | 队友间**直接互发消息、自主讨论** |
| **任务管理** | 主代理强制分配 | 共享列表、可自主认领、支持依赖 |
| **适用场景** | 单一专注、结果导向任务 | 复杂项目、多角色协作、并行开发 |
| **Token成本** | 较低（结果合并） | 较高（每个队友独立计费） |

**一句话总结**：Subagent 是“主仆分工”，Agent Teams 是“**平级团队协作**”。

## 二、启用 Agent Teams（必须手动开启）
### 1. 版本要求
- Claude Code **v2.1.32+**（2026年2月后版本）
- 订阅：**Pro / Max 计划**（免费版不支持）

### 2. 启用方法（3种任选）
#### 方法1：配置文件（推荐，永久生效）
```bash
# 用户级（全局）
~/.claude/settings.json

# 项目级（当前项目）
项目根目录/.claude/settings.json
```
添加：
```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```


#### 方法2：环境变量（临时）
```bash
# macOS/Linux
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude

# Windows PowerShell
$env:CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS = "1"
claude
```


#### 方法3：启动参数
```bash
claude --env CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
```


### 3. 验证启用
发送：
```
帮我验证 Agent Teams 是否启用，创建测试团队
```
出现“团队创建”“队友生成”即成功。

## 三、支持哪些模型？（必须 Claude 3 系列）
### 1. 官方支持模型（仅 Anthropic 原生）
**仅支持 Claude 3 全系列**，不支持 Claude 2 / 2.1 / 旧版，**不支持国产模型替换**：
- **Claude 3 Opus**（最强）：架构设计、复杂决策、代码审查、统筹领队
- **Claude 3 Sonnet**（平衡）：**主力开发、业务代码、调试、测试**（最常用）
- **Claude 3 Haiku**（轻量）：信息查询、简单搜索、文档整理、低成本任务

### 2. 模型分配最佳实践（成本+效果）
- **Team Lead（领队）**：**Opus**（统筹、决策、质量把关）
- **开发/核心队友**：**Sonnet**（平衡能力与成本）
- **辅助/查询角色**：**Haiku**（低成本、快速响应）

### 3. 重要限制（关键）
- **不支持国产模型**（通义/GLM/DeepSeek等）：Agent Teams 是**原生闭源实验功能**，依赖 Anthropic 底层多实例调度，**无法通过 LiteLLM/One-API 替换**
- 每个 Teammate 是**完整独立计费实例**：3个队友 ≈ 4倍 Token 消耗（1领队+3队友）

## 四、创建与使用：3步组建AI开发团队
### 1. 自然语言创建（最简单）
直接发送指令，Claude 自动拆角色、建团队、派任务：
```
组建一个全栈开发团队，开发用户认证系统：
- 1名 架构师（Opus）：设计整体方案、数据库、接口规范
- 2名 后端工程师（Sonnet）：实现Java接口、JWT、权限控制
- 1名 前端工程师（Sonnet）：React登录/注册页面
- 1名 测试工程师（Haiku）：生成单元测试、接口测试
所有修改前需我批准，并行开发
```


### 2. 显式指定模型（精准控制）
```
创建团队：
- 领队：使用 Opus，只统筹不写代码
- 队友1：架构师（Sonnet），设计数据库
- 队友2：后端开发（Sonnet），写业务逻辑
- 队友3：代码审查（Opus），只读，审核所有变更
```


### 3. 委派模式（领队只协调不干活）
- 开启：`Shift+Tab`（循环切换到 Delegate Mode）
- 效果：领队**仅分配任务、协调、审批**，不亲自写代码

## 五、界面与操作：两种显示模式
### 1. In-process（内嵌模式，默认）
- 所有队友在**同一终端**，用 `Shift+Up/Down` 选择
- 按 `Enter` 进入队友会话，`Esc` 打断当前任务
- 任务列表：`Ctrl+T` 开关

### 2. Split Panes（分屏模式，tmux/iTerm2）
- 每个队友**独立窗格**，可同时查看所有进度
- 适合多任务并行监控、复杂项目协作

### 3. 常用快捷键
- `Shift+Up/Down`：选择队友
- `Shift+Tab`：切换委派模式
- `Ctrl+T`：显示/隐藏任务列表
- `Esc`：中断队友当前任务

## 六、实际运行表现（真实协作流程）
### 1. 典型协作流程（自动执行）
1. **需求拆解**：领队（Opus）分析需求 → 拆成模块 → 定义角色 → 生成任务列表
2. **任务分配**：队友自主认领或领队指派 → 状态变为「进行中」
3. **自主沟通**：
   - 后端问前端：“接口字段如何定义？” → 直接发消息，无需你介入
   - 测试通知开发：“发现3个Bug，路径如下”
4. **审批机制**（可选）：
   ```
   所有代码修改前必须经我批准
   ```
   队友先出**只读计划** → 你确认 → 才执行写入
5. **进度同步**：任务完成 → 状态更新 → 依赖任务自动解锁
6. **结果合成**：全部完成 → 领队整合代码、文档、测试 → 交付完整项目

### 2. 直观表现（你会看到）
- 多AI同时输出日志、并行读/写文件
- 队友间互发消息：「队友A → 队友B：我已完成用户表，你可开发接口」
- 任务看板实时更新：待处理→进行中→已完成
- 冲突自动协调：多人改同一文件时，领队提醒、排队、合并

### 3. 效率提升
- **并行开发**：比单AI顺序快 **3–5倍**（同时做架构、后端、前端、测试）
- **减少干预**：AI自主沟通，你只需把控方向、审批关键节点

## 七、适用场景与避坑
### 1. 最佳场景
✅ **大型项目并行开发**（全栈、微服务、多模块）
✅ **多角度代码审查**（安全、性能、规范、可读性同时检查）
✅ **方案对撞/技术选型**（多个AI给出不同方案，择优）
✅ **测试+开发并行**（边写代码边生成测试）

### 2. 不适合场景
❌ **简单小功能**（单AI更快更省）
❌ **国产模型环境**（不支持替换，只能用 Claude 3）
❌ **极低预算**（多实例 Token 消耗高）

### 3. 常见问题
- **会话恢复失败**：实验功能，重启后团队可能丢失 → 及时保存配置
- **文件冲突**：避免多人同时改同一文件 → 明确文件归属
- **成本过高**：控制队友数量（**3–5个最佳**），简单任务用 Haiku

## 八、总结
### 核心要点
- **Agent Teams = 多AI并行团队**，独立实例、自主沟通、共享任务
- **仅支持 Claude 3（Opus/Sonnet/Haiku）**，**不支持国产模型**
- **Pro/Max 订阅+手动启用**，免费版不可用
- **表现**：多AI同时工作、自主沟通、并行开发、效率提升3–5倍

### 一句话建议
**复杂项目用 Agent Teams，简单任务用 Subagent**；国内用户需接受“**只能用 Claude 3、成本更高**”的前提，但换来真正AI团队级开发能力。
