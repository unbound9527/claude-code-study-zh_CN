# 项目永久记忆：CLAUDE.md 完全使用教学
**CLAUDE.md 是 Claude Code 的“项目大脑”与“永久记忆库”**，也是实现高效、稳定、可复用 AI 协作的核心基石。它能让 AI 每次会话都自动理解项目背景、遵守编码规范、记住关键决策，彻底告别重复解释、反复调教、上下文遗忘的低效困境。

本文从原理、创建、编写、维护、分层、实战到最佳实践，全面讲解如何用 CLAUDE.md 把 Claude Code 变成深度适配你项目的专属开发助手。

## 一、CLAUDE.md 是什么？（核心本质）
### 1. 定义
**CLAUDE.md** 是存放在项目根目录（或子目录）的 **Markdown 格式项目说明书**，Claude Code **每次启动新会话都会自动加载**，作为持久化上下文注入 AI 记忆。

### 2. 核心作用（为什么必须用）
- **永久记忆**：跨会话保存项目信息，关闭重开不丢失
- **统一标准**：团队协作时，AI 输出风格、规范、逻辑完全一致
- **减少重复**：不用每次都解释技术栈、架构、规范、命令
- **精准控制**：强制 AI 遵守规则、避开坑点、按指定流程开发
- **提升质量**：代码更贴合项目架构、更少 Bug、更易维护

### 3. 与对话上下文的区别
- **对话上下文**：临时记忆，会话结束即消失，易溢出、易污染
- **CLAUDE.md**：永久记忆，持久存储、自动加载、可控可维护

## 二、三步快速上手：创建与初始化
### 1. 自动初始化（推荐）
在 Claude Code 终端执行：
```bash
/init
```
- 自动扫描项目：技术栈、目录结构、依赖、常用命令
- 自动在**根目录**生成 `CLAUDE.md` 初始草稿
- 生成后**必须人工审核精简**（初始版偏全、偏冗余）



### 2. 手动创建（无 /init 时）
直接在项目根目录新建文件：
```
项目根目录/CLAUDE.md
```

### 3. 编辑与查看
- 命令行编辑：
  ```bash
  /memory  # 打开编辑器直接修改 CLAUDE.md
  ```
- 手动编辑：用 VS Code 等编辑器直接修改文件

## 一、技术栈（核心环境）
- 前端：React 18 + TypeScript + Ant Design + Vite
- 后端：Node.js + Express + TypeScript + JWT
- 数据库：MongoDB + Mongoose
- 工具：ESLint + Prettier + Jest + Docker

## 二、目录规范（路径规则）
- 前端：src/ 下按功能分模块（pages/components/hooks/utils）
- 后端：src/
  - routes/     API 路由
  - controllers/ 业务逻辑
  - models/     数据模型
  - middleware/ 中间件
  - utils/      工具函数
- 禁止：直接修改 src/generated/（自动生成目录）
- 测试：__tests__ 目录，与源码同级

## 三、编码规范（强制约束）
### 代码风格
- 命名：驼峰法（变量/函数）、帕斯卡（组件/类）、下划线（数据库字段）
- 缩进：4 空格，禁止 Tab
- 单行长度：≤ 120 字符
- 导出：优先具名导出，少用 default

### 必须遵循
- 所有接口必须加 JWT 校验（除登录/注册）
- 密码必须用 bcrypt 加密（禁止 MD5/SHA1）
- 错误返回统一格式：{ code: number, msg: string, data?: any }
- 关键函数、接口、复杂逻辑必须加注释

### 禁止行为
- 禁止硬编码密钥、IP、端口（用 .env）
- 禁止在生产代码留 console.log
- 禁止未校验直接操作数据库

## 四、高级特性：层级加载（全局/项目/子模块）
Claude Code 支持**多层级 CLAUDE.md**，按**就近优先**原则自动合并，实现精准管控：

### 1. 层级优先级（从高到低）
1. **子模块级**：`子目录/.claude/CLAUDE.md`（当前模块专属）
2. **项目级**：`项目根目录/CLAUDE.md`（全项目通用）
3. **用户级**：`~/.claude/CLAUDE.md`（个人所有项目通用）
4. **系统级**：`/etc/claude-code/CLAUDE.md`（团队/企业统一）

### 2. 典型结构（monorepo 推荐）
```
项目根/
├── CLAUDE.md               # 全局通用规范
├── frontend/
│   └── .claude/
│       └── CLAUDE.md       # 前端特有规则（覆盖全局）
└── backend/
    └── .claude/
        └── CLAUDE.md       # 后端特有规则（覆盖全局）
```

### 3. 使用场景
- **全局**：通用编码风格、禁止行为、基础命令
- **前端**：UI 框架规范、组件规则、打包配置
- **后端**：API 规范、数据库规则、安全约束
- **用户级**：个人偏好（语言、注释风格、回复方式）

## 五、编写黄金法则（高质量关键）
### 1. 指令明确：用强约束词
- ✅ **必须/禁止/优先/仅**（清晰可执行）
- ❌ **尽量/建议/最好/大概**（模糊无效）

### 2. 内容精简：只写核心
- **不写通用常识**（如 JS 基础语法）
- **不写详细文档**（放 docs/，CLAUDE.md 只加索引）
- **删除过时内容**（定期清理）
- **控制行数**：200–300 行（避免占满上下文）

### 3. 结构清晰：分级标题
- 用 `# / ## / ###` 分层，AI 易解析
- 模块间用 `---` 分隔，可读性强

### 4. 动态维护：随项目迭代
- **每完成重要任务**：更新「关键决策」
- **每踩一个坑**：添加到「项目坑点」
- **每改一次规范**：同步更新并提交 Git

## 六、实战工作流：CLAUDE.md 全生命周期
### Step 1：新项目初始化
1. 克隆/创建项目 → 打开 Claude Code
2. 执行 `/init` → 自动生成 CLAUDE.md
3. 按标准结构**精简、补充、修正**
4. 提交到 Git：`git add CLAUDE.md && git commit -m "feat: add CLAUDE.md"`

### Step 2：日常开发（自动生效）
1. 启动新会话 → **自动加载 CLAUDE.md**（无需手动操作）
2. 直接发指令：`实现用户注册接口` → AI 自动遵守所有规范
3. 完成任务 → 让 AI 总结决策：
   ```
   总结本次关键决策，更新到 CLAUDE.md 的「关键决策」
   ```
4. 手动确认更新 → 提交 Git

### Step 3：任务切换（干净上下文）
1. 执行 `/clear` → 清空临时对话记忆
2. 新任务自动加载最新 CLAUDE.md → 无干扰、无污染

### Step 4：团队协作（统一标准）
1. 团队共用一份 CLAUDE.md，提交到仓库
2. 新人加入 → 拉取代码 → 直接用 Claude Code → **零学习成本**
3. 规范变更 → 统一修改 CLAUDE.md → 全员同步

## 七、常见误区与避坑
### ❌ 误区 1：写得越长越全越好
- **真相**：越长越臃肿，AI 难聚焦、占上下文、响应慢
- **正确**：**极简核心**，只写 AI 必须知道的项目特有信息

### ❌ 误区 2：只初始化不维护
- **真相**：项目迭代后，CLAUDE.md 过时 → AI 按旧规则输出，错误频发
- **正确**：**动态更新**，随项目同步演进，定期 review

### ❌ 误区 3：不分层级，全写根文件
- **真相**：子模块规则被全局覆盖，无法精准管控
- **正确**：**分层设计**，全局通用 + 模块专属

### ❌ 误区 4：不提交 Git，仅本地存在
- **真相**：团队协作混乱、换设备丢失、无法回溯
- **正确**：**必须提交 Git**，作为项目核心配置文件管理
