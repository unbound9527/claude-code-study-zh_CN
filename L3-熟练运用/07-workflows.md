# 常见工作流

## 工作流是什么

完成特定任务的标准步骤。像做饭：备菜→切菜→炒菜→装盘。

## 代码审查

```
阅读代码 → 分析问题 → 提供建议 → 实施修改
```

## Bug 修复

```
描述错误 → 定位原因 → 修复代码 → 验证
```

## 学习新技能

```
概览 → 深入 → 实践 → 答疑 → 总结
```

## 写作

```
初稿 → 润色 → 结构 → 校对 → 完成
```

## 项目开发

```
需求 → 架构 → 实现 → 测试 → 文档
```

## 学习备考

```
整理知识点 → 出题 → 练习 → 批改 → 查漏补缺
```

## 设计自己的工作流

1. 识别重复步骤
2. 提取固定模式
3. 保存为模板
