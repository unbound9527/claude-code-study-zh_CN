# 迭代优化答案

## 什么是迭代

```
第一版 → 不满意 → 调整 → 第二版 → 还不够好 → 再调整 → ... → 满意
```

好结果往往需要多轮调整。

## 常见模式

**逐步完善**：方案 → 第三点展开 → 加时间表 → 检查预算

**风格调整**：写文章 → 太正式 → 轻松点 → 加幽默 → 精简

**重构改进**：代码 → 太复杂 → 简化 → 加注释 → 提效

## 调整指令

| 目的 | 说法 |
|------|------|
| 更简洁 | "精简到一半""一句话总结" |
| 更详细 | "展开说说""给更多例子" |
| 改风格 | "口语化""更专业" |
| 换角度 | "从成本角度考虑" |

## 调整层次

| 层次 | 内容 |
|------|------|
| 词汇 | 换词、换说法 |
| 句子 | 改句子结构 |
| 段落 | 调段落顺序 |
| 结构 | 改整体框架 |

## 有效 vs 无效迭代

| ❌ 无效 | ✅ 有效 |
|--------|--------|
| "再写一遍" | "第三段精简" |
| "重新回答" | "换大学生场景的例子" |
| "不对" | "按背景-问题-方案重组" |

## 纠正 AI 的错误

| 情况 | 怎么办 |
|------|--------|
| 答非所问 | 重新明确问题 |
| 信息错误 | 指出并要求核实 |
| 不完整 | 追问剩下的 |
| 太简单/太复杂 | 要求调整 |

**直接纠正**：
```
❌ "你好像搞错了"
✅ "第三点不对，爱因斯坦出生于1879年，请更正"
```

**说明你要什么**：
```
✅ "我要的是大学生平价穿搭，不是职场穿搭"
```
