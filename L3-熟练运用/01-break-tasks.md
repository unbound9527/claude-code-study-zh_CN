# 分解复杂任务

## 为什么要分解

| 大任务（难） | 小任务（易） |
|-------------|-------------|
| "写一本书" | "写第一章" |
| "装修房子" | "设计客厅" |
| "准备考试" | "整理知识点" |

**大任务 = 多个小任务的组合**

## 金字塔思维

```
顶层：最终目标
中层：主要步骤（3-5个）
底层：具体行动
```

## 需要分解的信号

- 问题太宽泛："怎么成功？"
- 涉及多个领域
- 步骤不明确
- 预期结果不具体

## 和 Claude Code 合作

```
❌ "帮我从零做小程序"
→ 太复杂，AI不知从何说起

✅ "做待办小程序，按顺序：
第一步：告诉我需要哪些步骤
第二步：设计数据库
第三步：写前端..."
```

## 模板

```
我要做 [目标]，请按步骤帮我：
第一步...
第二步...
第三步...
每完成一步等我确认后再继续。
```
