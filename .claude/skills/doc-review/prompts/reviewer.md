# 文档专业性核查 Prompt

## 任务

对 Markdown 文档进行专业性核查，检查技术准确性、内容完整性和受众匹配度。

## 核查规则

### 1. 技术准确性

#### 概念检查
- 核心理念念定义是否准确
- 与官方文档是否一致
- 没有事实性错误

#### 术语检查
- 术语使用是否符合通用惯例
- 是否存在术语混用
- 缩写首次出现是否有全称

### 2. 代码示例

#### 语法正确性
- 代码块是否有正确语言标记
- 语法是否正确
- 路径是否合理

#### API 匹配
- 使用的 API 是否与文档描述版本匹配
- 参数是否正确
- 返回值说明是否准确

#### 命令有效性
- 命令是否在当前版本可用
- 是否有已废弃命令
- 选项是否正确

### 3. 内容完整性

#### 结构检查
- 是否包含目标、概念、实践、练习、测验
- 各章节是否有实质性内容
- 是否有未完成的占位符

#### 示例充分性
- 关键概念是否有示例
- 示例是否具有代表性
- 边界情况是否有说明

### 4. 受众匹配度

#### 难度评估
- 内容难度是否与阶段匹配
- 新术语是否有解释
- 是否有必要的背景铺垫

#### 实用性
- 练习是否贴近实际使用场景
- 案例是否有参考价值
- 注意事项是否全面

## 实现步骤

1. 扫描目标目录下的所有 Markdown 文件
2. 对每个文件进行上述各项检查
3. 记录发现的问题和修改建议
4. 生成完整核查报告
