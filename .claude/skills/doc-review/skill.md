# 文档专业性核查 Skill

## 用途

核查 Markdown 文档的专业性和准确性：
- 技术概念正确性
- 代码示例可运行性
- 链接有效性
- 内容完整性
- 目标受众匹配度

## 使用方式

```
/review <文件路径或目录>
```

不指定路径时，默认检查 `docs/` 目录。

## 核查项目

### 1. 技术准确性
- 概念定义是否正确
- 技术术语使用是否准确
- 原理说明是否合理

### 2. 代码示例
- 语法正确性
- API 调用是否与文档版本匹配
- 命令是否过时

### 3. 内容完整性
- 章节是否有缺失
- 示例是否充分
- 练习题答案是否提供

### 4. 受众匹配度
- 难度是否与目标人群匹配
- 术语解释是否充分
- 是否有必要的背景知识铺垫
