# 格式规范验证 Prompt

## 任务

检查 Markdown 文件的格式规范。

## 检查项目

### 1. 分割线
- 检查是否使用 `---` 作为分割线
- 检测 `***` 或 `___` 等非标准分割线

### 2. 列表符号一致性
- 检查文件中列表符号是否一致（`-` 或 `*` 或 `+`）
- 混用视为问题

### 3. 代码块
- 检查是否使用 ``` 并标注语言
- 检测未闭合的代码块

### 4. 链接格式
- 检查 Markdown 链接格式 `[text](url)`
- 检测裸链接（如直接写 URL 而非 `[text](url)` 格式）

### 5. 强调和特殊符号
- 检查 `**粗体**` 和 `*斜体*` 是否成对
- 检测特殊字符使用（如全角空格）

## 输出格式

```markdown
### 格式规范检查结果

**扫描文件数**: N
**发现问题数**: M

#### 问题列表

| 文件 | 位置 | 类型 | 问题描述 |
|------|------|------|----------|
| xxx.md | 行 20 | 分割线 | 使用 `***` 而非 `---` |
| xxx.md | 行 35 | 列表 | 混用 `-` 和 `*` |
| xxx.md | 行 50 | 代码块 | 未闭合的代码块 |
```
