# Claude Code 清晰表达需求：精准指令实战指南
在 Claude Code 中，**清晰表达需求 = 完整背景 + 明确任务 + 严格约束 + 可验证结果**，核心是把 AI 当作“零背景、强执行、懂细节”的资深开发，不省略、不模糊、不假设。本文从原则、结构、模板、技巧、误区全流程讲解，帮你一次说清需求、少返工、高匹配。

---

## 一、核心原则（先懂再用）
### 1. 绝对具体，杜绝模糊
AI 不会读心，**所有细节必须显式写出**，禁用“优化、美化、完善、更好”等模糊词。
- ❌ 错误：“写个登录功能”“优化代码”
- ✅ 正确：“用 Kotlin+Retrofit 写登录接口，校验手机号格式，处理 400/401/500 错误，返回标准 JSON”

### 2. 完整上下文，不缺信息
Claude Code 能读项目，但**必须主动告知关键背景**：
- 技术栈（语言+框架+版本）、架构、目录规范
- 现有代码、依赖、接口、数据结构
- 项目规则、编码规范、禁忌、历史坑点

### 3. 强约束，明确边界
告诉 AI **必须做、禁止做、优先做**，划定执行范围：
- 必须：遵循 MVVM、空安全、异常捕获
- 禁止：修改 package.json、引入新库、硬编码密钥
- 优先：用协程、懒加载、组件复用

### 4. 可验证，结果可衡量
需求必须含**验收标准**，AI 可自检、你可快速判断：
- 验证：运行 `./gradlew test`、编译通过、无崩溃
- 结果：功能正常、符合规范、性能达标

### 5. 结构化，易解析
用标题、列表、代码块、XML 标签拆分信息，**长需求必结构化**。

---

## 二、需求五要素（必含）
### 1. 背景（Context）
- 项目：名称、类型、目标用户
- 技术栈：`Kotlin 1.9 + Compose 1.6 + MVVM + Room + Retrofit 2.9`
- 架构：`app/domain/data` 分层、模块化
- 规范：Material3、协程、空安全、注释规范

### 2. 任务（Task）
**明确动作+对象+范围**，一句话说清做什么：
- 编写：`src/ui/compose/LoginPage.kt` 登录页
- 修复：`MainActivity` 崩溃问题
- 优化：`Repo` 层 N+1 查询
- 审查：`api` 模块安全风险

### 3. 功能细节（Feature）
**做什么+输入+输出+流程**，越细越准：
- 输入：手机号（11位）、密码（≥8位）
- 流程：校验→请求接口→存储 Token→跳转首页
- 输出：`{code:200, msg:"ok", data:{token:"xxx"}}`

### 4. 约束（Constraints）
**必须/禁止/边界/性能/安全**，列清单：
- 必须：适配 Android 8+、深色模式、屏幕旋转
- 禁止：使用 `AsyncTask`、`Handler`、全局变量
- 边界：仅处理登录，不包含注册/找回密码
- 性能：接口超时 10s、避免内存泄漏

### 5. 验收（Success）
**如何算完成、如何验证、输出形式**：
- 验证：编译通过、预览正常、点击有效、无报错
- 输出：完整代码、注释、调用示例、测试用例

## 三、常见误区（避坑）
### ❌ 误区1：需求模糊，信息缺失
- 问题：AI 猜需求，输出偏离、反复返工
- 解决：**严格按五要素写**，不省略任何细节

### ❌ 误区2：一次多任务
- 问题：AI 顾此失彼、遗漏要点、逻辑混乱
- 解决：**一次一个核心任务**，复杂任务分多轮

### ❌ 误区3：不写约束，放任自由
- 问题：代码不符合规范、技术选型错误、引入风险
- 解决：**必须加约束清单**，明确允许/禁止/必须

### ❌ 误区4：否定句低效
- ❌ “不要用 Java 写”“不要写冗余代码”
- ✅ “用 Kotlin 编写”“代码精简、无冗余逻辑”

### ❌ 误区5：不验证，生成即结束
- 问题：代码能跑但不符合标准、隐藏 Bug
- 解决：**需求必含验收步骤**，AI 自动验证

## 四、一句话总结
**Claude Code 清晰需求 = 背景 + 任务 + 功能 + 约束 + 验收**
把 AI 当执行者，**你定规则、给信息、验结果**，AI 精准执行、一次到位。

掌握这套方法，你的需求表达将**精准、高效、可控**，让 Claude Code 成为你最靠谱的开发助手，大幅提升编码效率。
