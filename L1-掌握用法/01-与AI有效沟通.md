# 与 AI 有效沟通

## AI 是什么

AI 不是真的"智能"，更像记忆力超强的图书馆管理员——根据学过的内容组织回答，不是有真正的理解。

Claude Code 通过工具调用"感知世界、执行动作"，不只是一个聊天机器人。

## 对话 vs 聊天

| 聊天 | Claude Code |
|------|-------------|
| 朋友记得上次说的 | 默认只记当前对话 |
| 朋友会猜你意思 | AI 只能按你说的理解 |
| 朋友会纠正你 | AI 尽量回答你 |

## 好问题 vs 坏问题

| 坏问题 | 好问题 | 为什么 |
|--------|--------|--------|
| "帮我看看" | "检查这段文字语法" | 太模糊 |
| "怎么做" | "做番茄钟 APP，告诉我步骤" | 缺背景 |
| "那个事" | "上周写的邮件再发一遍" | 指代不清 |

## 提问三法则

**1. 说清楚要什么**
```
❌ 帮我写东西
✅ 100字内写朋友圈文案，推荐新耳机
```

**2. 给背景**
```
❌ 这个方案怎么样
✅ 大学生校园卖零食，定价8元/包进价5元，合理吗
```

**3. 指定格式**
```
❌ 列几个学习方法
✅ 5个高效学习法，用表格，含方法名/步骤/适用场景
```

## 理解 AI 的回答

AI 回答可能有：

| 情况 | 怎么办 |
|------|--------|
| 正确且完整 | 采纳 |
| 正确但不完整 | 追问 |
| 部分错误 | 纠正 |
| 完全错误 | 重新问 |

### 警惕"幻觉"

AI 可能编造看似真实的信息。

**检验方法**：让 AI 提供信息来源，或用搜索引擎核实。

### 追问技巧

| 情况 | 问法 |
|------|------|
| 太笼统 | "第三步逻辑是什么？" |
| 不懂术语 | "这个词什么意思" |
| 要简化 | "用小学生能懂的话讲" |
| 要扩展 | "再举3个例子" |

### AI 的局限性

| 原本不知道 | 现在如何获取 |
|-----------|-------------|
| 今天的事 | WebFetch 获取 |
| 你的信息 | Memory 或对话中告诉它 |
| 实时数据 | MCP 工具 / WebFetch |
| 文件内容 | Read/Glob/Grep |
