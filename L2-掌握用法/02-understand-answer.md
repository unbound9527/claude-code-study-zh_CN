# 理解 AI 的回答

## 回答质量判断

AI 回答可能有：

| 情况 | 怎么办 |
|------|--------|
| 正确且完整 | 采纳 |
| 正确但不完整 | 追问 |
| 部分错误 | 纠正 |
| 完全错误 | 重新问 |

## 警惕"幻觉"

AI 可能编造看似真实的信息（幻觉）。

**检验方法**：
- 搜索引擎核实
- 让 AI 提供信息来源

## 追问技巧

| 情况 | 问法 |
|------|------|
| 太笼统 | "第三步逻辑是什么？" |
| 不懂术语 | "这个词什么意思" |
| 要简化 | "用小学生能懂的话讲" |
| 要扩展 | "再举3个例子" |
| 要替代 | "还有其他方法吗？" |

## AI 的局限性

| 原本不知道 | 现在如何获取 |
|-----------|-------------|
| 今天的事 | WebFetch 获取 |
| 你的信息 | Memory 或对话中告诉它 |
| 实时数据 | MCP 工具 / WebFetch |
| 文件内容 | Read/Glob/Grep |

## AI 会犯的错

| 错误 | 应对 |
|------|------|
| 编造事实 | 核实 |
| 推理错误 | 验算 |
| 偏见 | 保持警惕 |
