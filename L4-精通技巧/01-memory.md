# 记忆功能

Claude Code 的记忆系统让你跨对话保持上下文，避免重复解释。

---

## 记忆类型

| 类型 | 作用域 | 持久性 | 创建方式 |
|------|--------|--------|----------|
| **对话记忆** | 当前对话 | 会话结束消失 | 自动 |
| **用户级记忆** | 所有对话 | 永久 | `~/.claude/CLAUDE.md` |
| **项目级记忆** | 当前项目 | 永久 | 项目根目录 `CLAUDE.md` |
| **目录级记忆** | 特定目录 | 永久 | 目录内 `CLAUDE.md` |

---

## 用户级记忆

用户级记忆作用于所有对话，适合记录个人偏好和常用配置。

### 创建方式

在用户主目录创建 `~/.claude/CLAUDE.md`：

```bash
# macOS / Linux
mkdir -p ~/.claude
vim ~/.claude/CLAUDE.md

# Windows
# 在用户目录下创建 .claude\CLAUDE.md
```

### 示例模板

```markdown
# 用户记忆

## 基本信息
- 名字：[你的名字]
- 时区：Asia/Shanghai
- 常用语言：中文、English

## 工作偏好
- 代码风格：简洁、函数式优先
- 注释要求：重要逻辑必须注释
- 测试态度：TDD 优先

## 技术栈
- 主语言：Python、TypeScript
- 框架：FastAPI、React
- 工具：Docker、Kubernetes

## 特殊要求
- 敏感信息不写入文件
- 生产操作需二次确认
```

### 查看记忆

```
请问记住关于我的哪些？
```

Claude 会列出当前记忆内容。

### 更新记忆

```
请更新：主语言 Python → Go
```

### 删除记忆

```
请删除关于 [某个主题] 的记忆
```

## 项目级记忆

项目级记忆只作用于当前项目，适合团队共享项目规范。

### 创建方式

在项目根目录创建 `CLAUDE.md`：

```bash
touch CLAUDE.md
```

### 项目记忆内容

```markdown
# 项目说明

## 项目概况
- 名称：[项目名]
- 类型：Web 应用 / API 服务 / CLI 工具
- 简介：[一句话描述]

## 技术栈
- 前端：React 18 + TypeScript
- 后端：Go + Gin
- 数据库：PostgreSQL + Redis
- 部署：Docker + K8s

## 代码规范

### 命名规范
| 类型 | 规范 | 示例 |
|------|------|------|
| 变量 | camelCase | `userName` |
| 常量 | UPPER_SNAKE | `MAX_RETRY` |
| 类名 | PascalCase | `UserService` |
| 文件名 | kebab-case | `user-service.ts` |

### Git 规范
- Commit 信息使用中文
- 分支命名：`feature/xxx`、`fix/xxx`
- PR 需要 code review

### API 规范
- RESTful 设计
- 响应格式：`{ success, data, error }`
- 状态码：200 成功、400 参数错误、401 未授权、500 服务器错误

## 常用命令

```bash
# 开发
npm run dev

# 测试
npm test

# 构建
npm run build

# 部署
./deploy.sh production
```

## 注意事项
- 敏感配置在 `.env.local`
- 定期执行 `npm update`
- 生产环境禁止 console.log
```

## 目录级记忆

目录级记忆作用于特定子目录，适合为不同模块设置不同规则。

### 创建方式

在目标目录内创建 `CLAUDE.md`：

```
src/
├── api/
│   ├── CLAUDE.md    # API 模块规范
│   └── routes/
├── components/
│   ├── CLAUDE.md    # 组件规范
│   └── ui/
└── utils/
    └── CLAUDE.md    # 工具函数规范
```

### 示例：API 模块规范

```markdown
# API 模块规范

## 职责
- 处理 HTTP 请求
- 参数校验
- 调用 service 层

## 约束
- 不直接操作数据库
- 所有响应统一封装
- 错误使用自定义异常类
```

## 记忆加载优先级

当多个记忆文件同时存在时，加载顺序如下：

```
1. 用户级记忆 (~/.claude/CLAUDE.md)
2. 项目级记忆 (项目根目录 CLAUDE.md)
3. 目录级记忆 (当前目录 CLAUDE.md)
```

**优先级**：目录级 > 项目级 > 用户级

同名规则时，更具体的规则优先。

---

## 最佳实践

### 1. 重要信息放项目级

团队共享的规范（如代码风格、Git 流程）放 `CLAUDE.md`，确保协作一致性。

### 2. 个人偏好放用户级

个人习惯（如常用语言、时区）放 `~/.claude/CLAUDE.md`。

### 3. 模块规则放目录级

不同模块有不同规范时，在对应目录创建 `CLAUDE.md`。

### 4. 定期清理记忆

```
请总结一下你记住的关于这个项目的内容
```

检查是否有过期或错误的信息，及时更新。

### 5. 避免信息冗余

不要在多个记忆文件中重复相同内容，只在最适合的位置放置。

## 常见问题

**Q：记忆文件放在哪里？**

A：取决于作用范围。用户级放 `~/.claude/CLAUDE.md`，项目级放项目根目录，目录级放对应目录。

**Q：新对话能记住之前的记忆吗？**

A：能。只要记忆文件存在，Claude 启动时会自动加载。

**Q：可以同时有多个项目记忆吗？**

A：可以。不同项目有不同的 `CLAUDE.md`，切换项目时 Claude 会自动加载当前项目的记忆。

**Q：记忆文件可以包含代码吗？**

A：可以。但建议保持简洁，详细的代码规范可以放在单独的文件中引用。

## 与 CLAUDE.md 的关系

`CLAUDE.md` 和对话记忆是两种不同的机制：

| 特性 | CLAUDE.md | 对话记忆 |
|------|-----------|----------|
| 持久性 | 永久 | 会话结束消失 |
| 作用范围 | 项目/目录 | 单次对话 |
| 更新方式 | 手动编辑 | "请记住" 指令 |
| 团队共享 | 支持 | 不支持 |

两者可以配合使用：
- `CLAUDE.md` 记录稳定的项目信息
- 对话记忆记录临时的一次性信息
