# 常见陷阱与避坑

总结了使用 Claude Code 过程中容易犯的错误和避坑方法。

---

## 技术陷阱

### 1. 上下文过长

**问题**：一次对话塞入过多代码和需求，上下文窗口耗尽，AI 开始"遗忘"早期信息。

**表现**：
- 后面的回答与前面的需求不一致
- AI 要求你"再描述一下背景"
- 响应越来越慢

**解决方法**：
- 定期用 `/clear` 清理上下文
- 复杂任务拆分成多个短对话
- 使用 CLAUDE.md 记录项目背景
- 关键信息在每次对话开头重复

### 2. 模糊指令

**问题**：指令不够明确，AI 自由发挥，结果跑偏。

**表现**：
- 生成的代码不是想要的
- AI 做了很多你没说"不要做"的事情
- 需要大量返工

**解决方法**：
- 具体说明：做什么 + 做到什么程度 + 约束条件
- 不做什么：明确列出边界
- 提供示例：给一个参考实现

### 3. 不验证 AI 的输出

**问题**：盲目相信 AI 生成的代码，直接用于生产。

**表现**：
- 语法错误没发现
- 逻辑漏洞没检测
- 安全问题被忽略

**解决方法**：
- 始终审查 AI 生成的代码
- 运行测试验证功能
- 关键操作先在测试环境验证
- AI 也会犯错，保持批判性思维

### 4. 忽略错误信息

**问题**：AI 给出的错误信息不仔细看，直接跳过。

**表现**：
- 同一个错误重复出现
- 问题越修越乱
- 浪费时间在错误方向上

**解决方法**：
- 仔细阅读 AI 的每一条分析
- 遇到不懂的错误，主动问 AI
- 记录常见错误，避免重复踩坑

## 工作流陷阱

### 5. 过度依赖 AI

**问题**：什么事都问 AI，连简单问题也不自己思考。

**表现**：
- 基础能力退化
- 遇到 AI 解决不了的问题就卡住
- 无法独立工作

**解决方法**：
- 简单问题自己解决，AI 用于复杂任务
- 学习 AI 的分析和解决思路
- 保留独立思考和动手能力

### 6. 不保存重要对话

**问题**：好的解决方案没有记录，下次遇到又要重新来。

**表现**：
- 同样的问题问了好几遍
- 好的 Prompt 没有积累
- 工作流无法复制

**解决方法**：
- 重要对话用 `/export` 导出保存
- 建立自己的 Prompt 模板库
- 记录有效的工作流程

### 7. 不利用 CLAUDE.md

**问题**：每个新对话都要重新解释项目背景。

**表现**：
- 重复的解释工作
- AI 对项目理解不一致
- 团队协作时风格不统一

**解决方法**：
- 项目根目录创建 CLAUDE.md
- 记录架构、规范、命令
- 团队成员共享 CLAUDE.md

### 8. 一次想做太多

**问题**：一个 Prompt 解决一整个大需求。

**表现**：
- 代码质量低
- 出错了难以定位
- 上下文爆炸

**解决方法**：
- 大任务拆成小步骤
- 每步完成后验证
- 用 `/plan` 规划整体方案

## 安全陷阱

### 9. 泄露敏感信息

**问题**：在对话中输入 API Key、密码、密钥等。

**表现**：
- 密钥进入对话历史
- 可能被记录和泄露
- 造成安全事故

**解决方法**：
- 使用环境变量代替硬编码
- 利用 MCP 配置敏感连接
- 敏感操作使用只读权限

### 10. 不设权限边界

**问题**：给 AI 过大权限，危险操作没有限制。

**表现**：
- AI 执行了 rm -rf /
- 生产环境被误操作
- 数据被意外删除

**解决方法**：
- 使用权限模式：`/permission-mode plan`
- 生产环境开启保护机制
- 危险操作前仔细确认

---

## 协作陷阱

### 11. 不 review AI 的改动

**问题**：AI 写的代码直接 commit，不做审查。

**表现**：
- 代码风格不一致
- Bug 进入代码库
- 安全漏洞被引入

**解决方法**：
- AI 改动必须经过人工 review
- 使用 Pull Request 流程
- 建立团队代码审查规范

### 12. 团队知识不共享

**问题**：个人用得好，但团队其他人不知道。

**表现**：
- 重复踩坑
- 最佳实践无法推广
- 团队效率参差不齐

**解决方法**：
- 分享有效的 Prompt 模板
- 记录使用技巧到团队 wiki
- 定期交流使用心得

---

## 避坑清单

每次使用 Claude Code 前，快速检查：

- [ ] 指令是否具体明确？
- [ ] 是否在泄露敏感信息？
- [ ] 大任务是否拆分了？
- [ ] 上下文是否过长了？
- [ ] 是否需要人工 review？
- [ ] 权限设置是否合理？
