# 高效使用模式

总结 Claude Code 的最佳使用模式，让你事半功倍。

---

## 核心原则

### 1. 明确任务边界

模糊的指令得到模糊的结果。明确定义：
- **要做什么**：实现功能 / 修复 Bug / 重构代码
- **约束条件**：性能要求 / 兼容版本 / 代码风格
- **交付标准**：功能验收 / 测试通过 / 文档齐全

### 2. 迭代式工作

不要试图一次完成所有事情：
```
需求 → 实现 → 验证 → 调整 → 完成
```

每轮迭代只解决一个小问题，降低出错概率。

### 3. 保持上下文简洁

Claude Code 的上下文窗口有限。保持对话聚焦：
- 一个任务完成后，再开始下一个
- 复杂任务拆分成多个简单步骤
- 定期用 `/clear` 清理无关上下文

## 场景化使用模式

### 快速编码

**适用场景**：日常编码、简单功能实现

**工作流**：
1. 明确需求（一句话描述）
2. AI 生成代码
3. 审查并提出修改意见
4. 迭代直到满意

**提示模板**：
```
实现用户登录功能：
- 使用 JWT
- 支持记住登录状态
- 密码加密存储
```

### 代码审查

**适用场景**：PR 审查、代码质量把控

**工作流**：
1. 提供代码上下文
2. 要求逐项检查（安全性、性能、可读性）
3. 获取改进建议
4. 选择性采纳

**提示模板**：
```
审查这个函数的安全性：
- 是否有 SQL 注入风险
- 是否有 XSS 风险
- 认证授权是否完善
```

### Bug 修复

**适用场景**：定位和修复问题

**工作流**：
1. 描述 Bug 现象
2. 提供相关代码和错误日志
3. 让 AI 分析可能原因
4. 实施修复
5. 验证修复效果

**提示模板**：
```
用户反馈：提交订单后页面显示"系统繁忙"
错误日志：[粘贴错误信息]
相关代码：[提供代码片段]
帮我定位问题并修复
```

### 重构优化

**适用场景**：改善代码结构、提升性能

**工作流**：
1. 说明重构目标（性能 / 可读性 / 可维护性）
2. 提供代码上下文
3. 让 AI 提出重构方案
4. 评审方案
5. 分步实施

**提示模板**：
```
这个函数的执行时间是 3 秒，需要优化到 300ms 以内：
[提供代码]
请分析瓶颈并给出优化方案
```

### 文档生成

**适用场景**：生成 API 文档、README、技术文档

**工作流**：
1. 说明文档类型和受众
2. 提供代码或功能描述
3. AI 生成初稿
4. 人工审核和补充

**提示模板**：
```
为这个 REST API 生成文档：
- API 端点：POST /api/orders
- 参数：user_id, items[], address
- 返回：order_id, status
受众：前端开发者
```

## 进阶技巧

### 利用 CLAUDE.md

在项目根目录创建 `CLAUDE.md`，记录：
- 项目架构和技术栈
- 代码规范和命名约定
- 常用命令和脚本
- 特殊业务规则

```markdown
# 项目说明

## 技术栈
- 前端：React 18 + TypeScript
- 后端：Node.js + Express
- 数据库：PostgreSQL

## 代码规范
- 使用 async/await，不使用 .then()
- 错误处理必须 try-catch
- 接口响应格式：{ success, data, error }
```

### 使用子代理处理复杂任务

对于需要深入分析的任务，使用子代理：
```
创建代码审查子代理，专门负责：
- 代码风格检查
- 性能问题识别
- 安全漏洞扫描
```

### 建立你的技能库

把常用的工作流封装成 Skill：
- 代码审查 Skill
- 文档生成 Skill
- 测试生成 Skill
