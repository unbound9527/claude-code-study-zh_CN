# 检查点与回滚

检查点让你保存会话状态并在需要时回滚，是 Claude Code 的"时光机"。

---

## 核心概念

| 概念 | 说明 |
|------|------|
| **Checkpoint** | 会话状态的快照，包含消息、文件、上下文 |
| **Rewind** | 回滚到之前的检查点 |
| **Branch Point** | 分支点，从这里可以探索不同方向 |

---

## 使用方式

### 键盘快捷键

连续按两次 `Esc` 打开检查点浏览器。

### 斜杠命令

```
/rewind
/checkpoint
```

---

## 回滚选项

回滚时提供 5 个选项：

| 选项 | 说明 |
|------|------|
| **恢复代码和对话** | 完全回滚到检查点 |
| **仅恢复对话** | 保持当前代码，回滚消息 |
| **仅恢复代码** | 保持对话历史，回滚文件修改 |
| **从这里总结** | 将后续对话压缩为摘要，原消息保留在记录中 |
| **算了** | 取消操作 |

---

## 自动检查点

Claude Code 自动创建检查点：

- **每个用户提示后**：新输入自动创建检查点
- **持久保存**：跨会话保存
- **30 天清理**：自动删除 30 天前的旧检查点

## 实战场景

### 探索不同方案

```
用户：添加缓存层

Claude：我来添加 Redis 缓存...
[创建检查点 A]

用户：算了，用内存缓存试试

Claude：[按 Esc+Esc 回滚到检查点 A]
[实现内存缓存，创建检查点 B]

用户：对比两个方案
```

### 从错误中恢复

```
用户：重构认证模块为 JWT

Claude：[进行了大量修改]

用户：等等，OAuth 集成坏了，回滚

Claude：[回滚到重构前的检查点]
```

### 安全实验

```
用户：试试用函数式风格重写

[实验前检查点]

Claude：[做出实验性修改]

用户：测试失败了，回滚

Claude：[回滚检查点]
[尝试另一种方案]
```

### 分支对比

```
用户：我想对比两种数据库设计
[记录初始检查点]

Claude：实现 Schema A...
[继续工作]

用户：回去试试方案 B
[Esc+Esc 回滚到初始检查点]

Claude：实现 Schema B...
```

## 工作流模式

### 分支探索策略

```
1. 初始实现 → 检查点 A
2. 尝试方案 1 → 检查点 B
3. 回滚到检查点 A
4. 尝试方案 2 → 检查点 C
5. 对比 B 和 C 的结果
6. 选择最佳方案继续
```

### 安全重构模式

```
1. 当前状态 → 自动检查点
2. 开始重构
3. 运行测试
4. 测试通过 → 继续
5. 测试失败 → 回滚到检查点
```

---

## 最佳实践

### 利用自动检查点

由于检查点自动创建，你可以专注于工作本身而无需手动保存。

### 回滚前先查看

回滚前查看可用检查点的详细信息：
- 时间戳
- 修改的文件
- 消息数量
- 使用的工具

### 选择合适的回滚方式

- 想完全重来 → 恢复代码和对话
- 想保持代码、换思路 → 仅恢复对话
- 想保留探索记录 → 从这里总结
